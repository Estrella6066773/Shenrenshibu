## 待细化追踪 · 使用约定

1. **收录范围**：任一库内尚未定案、未实现、与规则/实现/母本**有偏差**的项；不限于程序或单一模块。
2. **写入**：在 [待细化追踪.md](待细化追踪.md) 对应节增行；**来源**列链到具体文档或规则。
3. **关闭**：定案后回写来源文档 → 追踪表改状态 → 维护性说明移入 [待细化追踪-已完成.md](待细化追踪-已完成.md)（与主表分卷，避免膨胀）。
4. **优先级**：P0 阻塞提交/一致性/演示；P1 当前迭代；P2 排队；P3 可选。负责：**AI** / **协作** / **开发者** / **人工**（见主表「待办分级」）。
