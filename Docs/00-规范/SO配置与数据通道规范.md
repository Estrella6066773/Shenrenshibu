# SO 配置与数据通道规范

## 目标

统一《神人事部》的实现口径：**可调参数走 ScriptableObject（SO）或 `Assets/04_Data/` 配置，程序负责读取、校验、结算与来源记录**。

本规范适用于 `Docs/02-系统设计/` 与 `Docs/03-程序设计/` 的新增和修订文档。

## 核心原则

1. **参数配置化**
   - 员工标签、委托模板、分支树、压力事件、矛盾事件、邮件、聊天与对话等业务参数，必须通过 SO 或 `Assets/04_Data/` 配置。
   - 运行时代码不得写死日薪倍率、压力分段、委托分支权重、邮件事件键等可调参数。

2. **程序做数据通道**
   - 程序负责读取配置、校验缺失项、执行结算，并向 UI 或调试工具提供结果。
   - UI 只展示结果与来源，不重复实现业务结算。

3. **来源可追溯**
   - 数值修正应保留来源，例如员工标签、压力事件、矛盾事件、委托分支或编辑器配置。
   - 需要给玩家展示的 Buff、效率、扣薪、事件后果，不得只在 UI 层硬编码说明文字。

4. **目录统一**
   - 正式运行时数据以 `Assets/04_Data/` 为主目录。
   - 编辑器菜单创建的新资产应落入 `04_Data` 下对应模块目录。

5. **模块边界清晰**
   - Employee 负责员工标签与 Buff 聚合。
   - Assignment 负责委托实例、分支树与任务内效果。
   - Stress / Conflict 负责各自事件状态机与桥接消息。
   - MessageSystem / DialogSystem 负责消息、邮件、聊天、对话播放配置与历史状态。

## 禁止事项

- 在运行时代码中用员工显示名、委托显示名或邮件标题作为稳定主键。
- 在 UI 层自行计算日薪、效率、压力分段、分支结果等业务数值。
- 在模块内另建临时表绕过 `Assets/04_Data/` 的正式配置。
- 让编辑器菜单默认把正式 SO 创建到 `Assets/00_Data/`。
- 把剧情剧透层信息写入程序配置字段；剧情文稿与设定母本按设定库规则维护。

## 必备设计要素

新增或修订一个 SO 配置时，程序设计文档至少写清：

| 要素 | 说明 |
|------|------|
| 稳定 ID | 存档、消息、桥接引用使用的键 |
| 展示字段 | 面向玩家显示的名称、描述、图标等 |
| 业务参数 | 参与结算的数值、权重、阈值、标签或条件 |
| 引用关系 | 指向员工、委托、邮件、对话、设定条目的方式 |
| 默认值 | 缺省配置的处理方式 |
| 来源记录 | 运行时快照如何保留贡献来源 |

## 典型通道

| 通道 | 配置来源 | 程序入口 | 消费方 |
|------|----------|----------|--------|
| 标签 Buff | `EmployeeTagSO`、`EmployeeTagDatabaseSO` | `TagBuffAggregator` | 经济、委托、压力、UI |
| 委托分支 | `AssignmentTemplateSO`、`AssignmentBranchTreeSO` | `AssignmentController`、规则解析器 | 委托运行时、消息、压力、矛盾 |
| 压力事件 | `StressConfigSO`、`StressEventsCatalogSO` | `StressSystem.ApplyStressDelta` | 员工状态、邮件、委托 Scoped 效果 |
| 矛盾事件 | Conflict 配置 SO | `ConflictSystemService` | 委托、员工状态、邮件 |
| 消息与对话 | 频道、角色、邮件、剧情、对话配置 | MessageSystem、DialogSystem | UI、剧情历史、存档 |

## 文档落地要求

1. 系统设计说明玩家可感知的规则、验收口径与可调范围。
2. 程序设计的数据字典说明 SO 字段、引用关系与运行时快照。
3. 实现文档说明当前代码入口、模块边界与资产路径。
4. 交付文档或 UAT 说明如何校验关键配置存在且路径正确。
