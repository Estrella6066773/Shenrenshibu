## 第一条　适用范围

**禁止写入级别**的定案（作者口头定案且**暂时不准写入** `真相/`、`玩家投放/`、`剧情文稿/`）：

- **正文**仅存工程根 **`Docs-local/`** 对应子目录；
- **不进 Git、不进 UVC**；
- Git 内**只**保留本说明、 [待细化追踪](待细化追踪.md) 的编号与状态，**不得**复述定案细节。

## 第二条　本地根目录

```
神人事部/
└── Docs-local/          ← 整目录忽略（Git + UVC）
    ├── README.md
    └── 禁写层/
        ├── INDEX.md
        └── SET-EMB-{NNN}-{中文简名}.md
```

克隆仓库后若需协作禁写层，由作者**离线**提供 `Docs-local/` 目录；**不要**尝试 `git add Docs-local/`。

## 第三条　编号与文件名前缀（登记表）

| 类型 | 编号前缀 | 文件名前缀 | 本地子目录 | `visibility` | 登记 |
|------|----------|------------|------------|--------------|------|
| 设定禁写定案 | `SET-EMB-{NNN}` | `SET-EMB-{NNN}-` | `Docs-local/禁写层/` | `embargoed-local` | **不**写入 [id-registry.yaml](../设定设计/管理规则/id-registry.yaml) |

- `{NNN}`：三位数字，从 `001` 起；
- 文件名示例：`SET-EMB-001-林杜第三重身份.md`；
- 新增类型时：在本表增行，并在 `.gitignore` / `ignore.conf` **前缀屏蔽表**同步增行（见下节）。

## 第四条　版本控制屏蔽（目录 + 前缀双保险）

与 [00-工程集成说明](../00-工程集成说明.md) 一致：

| 规则 | `.gitignore`（Git） | `ignore.conf`（UVC） | 含义 |
|------|---------------------|----------------------|------|
| `/Docs-local/` | 是 | 是 | 本地保密根目录整树忽略 |
| `**/SET-EMB-[0-9]*` | 是 | 是 | 以 `SET-EMB-` + 数字开头的文件，**无论落在哪条路径**均不提交（防误放在 `Docs/`） |

`Docs/`、`.github/`、`Scripts/` 等 **Git 区**由 `.gitignore` 与 UVC 的 `ignore.conf` 分别屏蔽，**不要求**与上表逐条双写。UVC 只管程序侧（`Assets/`、`ProjectSettings/`、`Packages/`）。

**说明**：模板、说明类文件**不得**使用上表中的**屏蔽前缀**（例如用 `模板-禁写层条目.md`，不用 `SET-EMB-001-…` 命名模板）。

本地自检（可选）：

```powershell
python Scripts/verify-local-secrets-guard.py
```

## 第五条　Frontmatter（本地正文）

```yaml
id: SET-EMB-001
title: 林杜第三重身份
status: embargoed | lifted
visibility: embargoed-local
embargo_scope: 全设定库正文、CHR 百科、PCM、投放
related:
  - SET-REV-003
  - SET-REV-014
```

| 字段 | 必填 | 说明 |
|------|------|------|
| `id` | 是 | 与编号前缀一致 |
| `title` | 是 | 中文标题 |
| `status` | 是 | `embargoed` 禁写中；`lifted` 已解禁（正文可归档删除） |
| `visibility` | 是 | 固定 `embargoed-local` |
| `embargo_scope` | 是 | 不得写入的 Git 内路径范围（自然语言） |
| `related` | 否 | 关联的 `SET-REV-*` 或母本 `id` |

正文结构见 [\_templates/禁写层/模板-禁写层条目.md](../_templates/禁写层/模板-禁写层条目.md)。

## 第六条　Git 内允许登记的内容

## 第八条　新建条目流程

1. 在本地 `Docs-local/禁写层/` 复制 [模板-禁写层条目.md](../_templates/禁写层/模板-禁写层条目.md)，按 `SET-EMB-{NNN}-` 命名；
2. 更新本地 `Docs-local/禁写层/INDEX.md`；
3. 在本文件 §第七条 与 [待细化追踪](待细化追踪.md) 增一行（**仅**编号、标题、状态、禁写范围）；
4. 运行 `python Scripts/verify-local-secrets-guard.py`，确认 Git 工作区无屏蔽前缀文件；
5. **不要** `git add Docs-local/` 或任何 `SET-EMB-[0-9]*` 文件。

## 第九条　解禁流程

1. 作者明示解禁并指定目标层级（母本 `id`、投放版本等）；
2. 将定案**改写**进 `真相/`（或约定路径），走 [id-registry](../设定设计/管理规则/id-registry.yaml) 与投放升版；
3. 本地条目 `status: lifted`，正文移入归档或删除；
4. 本文件索引与 [待细化追踪](待细化追踪.md) 改状态 → 闭环移入 [待细化追踪-已完成](待细化追踪-已完成.md)；
5. 更新 [02-可见性与剧透分级](../设定设计/管理规则/02-可见性与剧透分级.md) 投放节奏。
