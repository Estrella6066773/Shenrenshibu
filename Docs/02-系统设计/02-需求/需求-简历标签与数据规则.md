## 1. 背景与目标

- **问题**：思维导图 C 区「人群互斥筛选」「隐藏标签」「排序」在代码侧部分存在，但 `REQ-DAT-002` 仅保证 tagId 唯一，缺互斥表、隐藏与排序的验收条文。
- **目标**：补齐简历入职、标签互斥、隐藏标签、列表排序等 REQ，与实现对齐或标为待实现。

## 2. 范围

- **纳入范围**：简历→员工卡入职、标签互斥组、隐藏标签展示规则、简历列表排序/伪冒泡周期。
- **不纳入范围**：证件照生成（见 [需求-证件照生成](需求-证件照生成.md)）。

## 3. 功能需求

| ID | 描述 | 优先级 | 验收要点 |
|----|------|--------|----------|
| REQ-RSM-005 | **入职**：简历拖入或确认后生成员工卡实例，字段从简历拷贝 | P1 | 冒烟：1 份简历→1 名员工；重复入职策略（拒绝/覆盖）文档化 |
| REQ-RSM-006 | **隐藏标签**：部分标签对玩家默认不可见，满足条件后揭示 | P1 | 至少 1 个隐藏标签用例；存档保留揭示状态 |
| REQ-RSM-007 | **排序**：简历列表支持策划定义的排序键与刷新周期（含「伪冒泡」） | P2 | 排序结果可复现；读档后顺序一致 |
| REQ-DAT-005 | **人群互斥**：互斥组表（如「人群 A」与「人群 B」不可同选） | P1 | 配置表可维护；筛选 UI 禁用互斥项 |
| REQ-EMP-005 | 员工卡**标签筛选**遵守互斥表（与 REQ-DAT-005 同一数据源） | P1 | 与简历筛选规则一致 |
| REQ-EMP-006 | 员工卡展示**隐藏标签**规则与 REQ-RSM-006 一致 | P1 | 揭示前后 UI 差异可测 |
| REQ-EMP-007 | **工作状态**枚举与任务进度联动表（工/事/待命/休整） | P1 | 与委托 Tick 状态一致；见 [需求-委托系统](需求-委托系统.md) |
