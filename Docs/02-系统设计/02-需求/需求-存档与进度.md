## 1. 背景与目标

- **问题**：各玩法模块（委托、消息、经济、矛盾等）已有分散存档字段，缺**总编排**与玩家可理解的「进度」语义。
- **目标**：定义游戏存档的组成、读写时机、槽位策略，并与 [需求-底部导航](需求-底部导航.md) 存档管理入口对齐。

## 2. 范围

- **纳入范围**：**存档集**与子存档、主存档指针、存档快照字段清单、读档恢复顺序、与 `save_location` 预留、手动/自动子档语义。
- **不纳入范围**：云存档、加密、版本迁移（0.x 不维护双轨）。

## 3. 功能需求

| ID | 描述 | 优先级 | 验收要点 |
|----|------|--------|----------|
| REQ-SAV-001 | 提供**统一存档编排**文档化契约：各模块导出/导入钩子清单与调用顺序 | P0 | 追溯矩阵可映射到具体 `*SaveCodec` / 行为 |
| REQ-SAV-002 | 读档时按约定顺序恢复：经济 → 员工 → 委托 → 消息 → 矛盾/压力 | P1 | 冒烟测试读档后核心循环可继续 |
| REQ-SAV-003 | 存档集内手动/自动子存档分离；每次保存**追加**子档并更新主存档 | P1 | 快速存档与自动保存均追加；历史子档保留 |
| REQ-SAV-004 | `schema_version` 不一致时拒读并提示玩家 | P1 | 与程序设计存档校验 UX 一致 |

## 4. 模块存档钩子清单（REQ-SAV-001）

读档顺序见 REQ-SAV-002；下表为各模块**导出/导入**责任划分（具体类型名以 `实现-*` 与代码为准）。

| 顺序 | 模块键 | 实现文档 | 存档载荷（摘要） | 备注 |
|:----:|--------|----------|------------------|------|
| 1 | PlayerEconomy | `实现-玩家经济与日历时钟.md` | 日历时钟、资金、`PlayerEconomySaveModels` | 日结状态 |
| 2 | Employee | `实现-员工与标签Buff.md` | 员工实例、标签、状态 | 与简历池分离 |
| 3 | Assignment | `实现-委托系统.md` | 活跃委托、阶段、分支进度 | 含回放字段 |
| 4 | MessageSystem | `实现-消息系统.md` | 聊天历史、频道归档、运行时收件箱 | `ChatHistoryStorageService` 等 |
| 5 | Conflict | `实现-矛盾系统.md` | 活跃/历史案件 | `Save/*` |
| 6 | Stress | `实现-压力系统.md` | `StressSaveData`、Tier、活跃事件 | 导出/导入 API |
| 7 | DialogSystem | `实现-对话播放系统.md` | 会话记忆、已读/已选节点 | 与 REQ-DLG-002 对齐 |
| 8 | StoryProgress | `需求-剧情进度.md` | `story_progress` 字段（待实现） | REQ-PRG-001 |
| — | Resume / IdleArea / Portrait | 各 `实现-*` | UI 布局或池状态（若纳入存档） | 首版可选子集 |

**存档集与编排层**：主菜单「存档管理」负责集/子档 CRUD；游戏内「快速存档」向活动集追加手动子档；`schema_version` 见 REQ-SAV-004。

### 存档集术语

| 标准名 | 说明 |
|--------|------|
| 存档集 | 玩家可见的一条游戏线；列表一块 |
| 子存档 | 集内单次保存快照 |
| 主存档 | 集内当前生效指针（最新有效子存档） |

## 5. 非功能需求

- 写盘：异步、tick 边界采集（见各 `实现-*` 非功能段）。
- 兼容性：0.x 仅 forward，不维护旧档迁移。
