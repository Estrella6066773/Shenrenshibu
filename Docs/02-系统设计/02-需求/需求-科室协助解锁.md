# 需求-科室协助解锁

| 字段 | 值 |

| 模块键 | DepartmentUnlock |
| 版本 | 0.1-draft |

## 1. 背景与目标

- **问题**：思维导图 D11~D13「协助三科解锁」在需求层为空白；设定上四科/三科协助与主线相关，但玩法 gates 未写 REQ。
- **目标**：为「协助人事部下属科室」类玩法提供解锁 flag 与委托支线契约，**待策划确认是否仍保留**后细化验收。

## 2. 范围

- **纳入范围**：科室协助解锁 flags、与剧情进度/委托奖励的关联、三科（或策划最终数量）协助玩法的入口条件。
- **不纳入范围**：四科科长人设正文（见 `04-设定/真相/10-百科/人物/`）。

## 3. 功能需求

| ID | 描述 | 优先级 | 验收要点 |
|----|------|--------|----------|
| REQ-UNL-001 | 维护**科室协助解锁 flags**（可序列化，纳入 REQ-SAV / REQ-PRG） | P2 | 读档后 flags 一致 |
| REQ-UNL-002 | 每科室（策划确认名单）对应**协助委托模板**或支线入口；未解锁时不可接取 | P2 | 至少 1 科冒烟：锁→解锁→可接 |
| REQ-UNL-003 | 解锁条件可来自：剧情进度（REQ-PRG）、主线节拍、或指定委托完成 | P2 | 条件表由策划维护；工程可配置 |

## 4. 首版定案（协作草案，2026-06-13）

| 项 | 定案 |
|----|------|
| **保留 scope** | 导图 D11~D13 **保留**；玩法映射为协助 **财务科、人力科、防卫科** 解锁（服务科为主线默认开放） |
| flag 命名建议 | `unl_finance_assist`、`unl_hr_assist`、`unl_defense_assist` |
| 解锁来源 | 主线节拍 + REQ-PRG 进度；详见 [PCM-01](../../04-设定/真相/30-主角与主线/PCM-01-开局主线剧情流程.md) |

> 若策划砍 scope，将本节标 `deprecated` 并更新追溯矩阵 `DepartmentUnlock` 行。

## 5. 策划待定项

| 项 | 说明 |
|----|------|
| 是否保留 | **首版保留**三科协助；见 §4 |
| 科室名单 | 财务科、人力科、防卫科（服务科不纳入 UNL gates） |
