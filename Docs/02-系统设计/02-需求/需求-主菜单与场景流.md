## 1. 背景与目标

- **问题**：思维导图要求独立主界面（开始游戏、继续游戏、设置）；当前工程以**游戏内**底部导航为主入口，缺 Title 场景与一键继续流程的需求锚点。
- **目标**：明确产品是否保留独立主菜单，或采纳「主场景直接 Play + 游戏内导航」策略，并写清验收口径。

## 2. 范围

- **纳入范围**：开始游戏、继续游戏、设置、退出游戏的入口定义与场景流。
- **Out Scope**：多存档云同步、账号系统。

## 3. 功能需求

| ID | 描述 | 优先级 | 验收要点 |
|----|------|--------|----------|
| REQ-MENU-001 | **开始游戏**：新游戏或选槽开局的路径（Title 场景或主场景等价流程） | P1 | 玩家从零可进入可玩状态 |
| REQ-MENU-002 | **继续游戏**：一键读取**最近**有效存档并恢复 | P1 | 少于 3 步到达游戏内场景；与 REQ-SAV 编排一致 |
| REQ-MENU-003 | **设置**：音量、分辨率等范围（可委托 REQ-NAV 子页） | P2 | 与 `实现-底部导航` 设置子页或 Title 设置页一致 |
| REQ-MENU-004 | **退出游戏** | P0 | 已有 REQ-NAV-002；本 REQ 引用不重复 |
| REQ-MENU-005 | **存档管理**：主菜单打开存档管理预制体 | P1 | 存档集列表、创建、标星、备份、删除 |
| REQ-MENU-006 | **关于**：仅 `IAboutPanelHost` 接口，首版无内容面板 | P2 | 调用不崩溃 |
| REQ-SCENE-001 | `主菜单` 为 Build index 0 启动场景 | P0 | Editor Play 从主菜单启动 |
| REQ-SCENE-002 | 场景切换经 `SceneFlowCoordinator` | P1 | 无散落 `LoadScene` 硬编码 |
| REQ-SCENE-003 | Editor 提供 Build 同步与场景校验菜单 | P2 | 菜单可执行、Console 输出 |

## 4. 产品决策

| 选项 | 说明 | 状态 |
|------|------|------|
| A. 独立 Title 场景 | `主菜单.unity` 为启动场景；继续/新游戏/存档管理/设置 | **已定（2026-07-02）** |
| B. 主场景 + 游戏内导航 | 主场景直接 Play；由底部导航承担 | 废止（首版 B 已升级） |

**验收口径（选项 A）**：

- **REQ-MENU-001**：「新游戏」= 创建存档集并进入主场景可玩态。
- **REQ-MENU-002**：「继续游戏」= 读取最近有效存档集**主存档**，一键进主场景。
- **REQ-MENU-003**：设置面板预制体，主菜单与游戏内可共用。
- **REQ-MENU-005**：存档管理面板支持存档集与子存档操作（见 REQ-SAV-005~008）。
