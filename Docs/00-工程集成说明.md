# Unity 工程集成说明

《神人事部》Unity 工程与文档的协作方式（对齐机魂）。

## 角色划分

| 载体 | 版本控制 | 职责 |
|------|----------|------|
| **神人事部（Unity 根）** | Git | `Docs/`、`Scripts/`、`.github/`、`.gitignore` 等 |
| **`Assets/`、`ProjectSettings/`** | UVC | 代码、场景、SO、美术 |

## 目录

```
神人事部/
├── Docs/
│   ├── 系统设计/
│   ├── 程序设计/
│   └── 设定设计/
├── .github/workflows/
└── Scripts/Get-DocsDate.ps1
```

## 改文档后的习惯

1. 在 **`Docs/`** 对应三分库内修改并提交。  
2. 在 **Unity 根目录** `git commit` / `git push`（与机魂相同，无子模块指针步骤）。  
3. 若改行为：同步 `系统设计/02-需求` 与 `程序设计/04-实现`，并更新 `程序设计/05-交付/交付-追溯矩阵.md`。  
4. 「最后更新」：在工程根运行 `powershell -NoProfile -File Scripts/Get-DocsDate.ps1`。
