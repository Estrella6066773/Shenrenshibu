# Unity 工程集成说明

## 版本控制原则（必读）

| 范围 | 工具 | 说明 |
|------|------|------|
| **`Docs/`** | **Git** | 唯一 GitHub 仓库：**[ShenrenshibuStoryLib](https://github.com/Estrella6066773/ShenrenshibuStoryLib)** |
| **`Docs-local/`** | **无** | 作者禁写层等本地文稿；**不进 Git、不进 UVC**（见 [00-规范/禁写层-本地说明.md](00-规范/禁写层-本地说明.md)） |
| **`Assets/`、`ProjectSettings/`、`Packages/`** | **UVC** | 程序与 Unity 资源，**不进 Git** |

- **本地 Git 根目录**：`神人事部/`（Unity 工程根）  
- **远程仓库**：`git@github.com:Estrella6066773/ShenrenshibuStoryLib.git`（**不是**新仓库，仍是原 StoryLib）  
- Git **只提交**文档与配套配置（`Docs/`、`.github/`、`Scripts/` 等）；程序与 Unity 资源由 UVC 管理。

`.gitignore` 排除 `Assets/`、`ProjectSettings/`、`Packages/`、`Docs-local/` 及前缀 `SET-EMB-[0-9]*`。  
`ignore.conf` **仅**用于防止 UVC 误纳入 Git 区与本地专用目录（见 [禁写层-本地说明](00-规范/禁写层-本地说明.md) §第四条）；**不要**把 Git 与 UVC 的规则混为同一套镜像。

## 目录

```
神人事部/                          ← git 根（remote = ShenrenshibuStoryLib）
├── Docs/                          ← 文档正文（系统设计 / 程序设计 / 设定设计）
├── Docs-local/                    ← 本地禁写层等（Git / UVC 均忽略）
├── .github/workflows/
├── Scripts/Get-DocsDate.ps1
├── Assets/                        ← UVC
├── ProjectSettings/               ← UVC
└── Packages/                      ← UVC（manifest / lock；Git 忽略）
```

## 日常操作

```powershell
cd "d:\Unity\神人事部"

git status
git add Docs/
git commit -m "docs: …"
git push -u origin main
```

```powershell
powershell -NoProfile -File Scripts\Get-DocsDate.ps1
```

**不要** `git add Assets/` 或 `ProjectSettings/`。

## 克隆（仅文档库）

若只要文档、不要整个 Unity 工程：

```bash
git clone git@github.com:Estrella6066773/ShenrenshibuStoryLib.git
# 仓库内含 Docs/ 与 CI；放入 Unity 工程时对应 神人事部/Docs/
```

## 历史说明

叙事与工程文档始终在 **ShenrenshibuStoryLib** 同一仓库、同一提交历史上扩展；本地在 Unity 根检出，`Docs/` 为机魂式三分库结构。
