# 命名、文件与标识符规则

## 第一条　标识符格式

设定条目的稳定标识符格式为：`前缀-主体`，全大写 ASCII，示例如下。

| 前缀 | 含义 | 示例 |
|------|------|------|
| WLD | 世界基石 | [WLD-00](../真相/00-基石/WLD-00-世界纪年与治理总纲.md) |
| ORG | 组织与势力 | [ORG-HELIOS](../真相/10-百科/组织/赫利俄斯/ORG-HELIOS-赫利俄斯集团.md) |
| PLC | 地点与空间类型 | [PLC-DISTRICT](../真相/10-百科/地点/PLC-DISTRICT-街区分层与层级-总述.md) |
| SYS | 见下表分两类存放（`id` 前缀不变） | [SYS-B9002](../真相/10-百科/系统/SYS-B9002-桥与玩家职能.md) |

**SYS 条目**：收录**人工智能系统**（管理者 AI 实例·桥谱系），见 [10-百科/系统/](../真相/10-百科/系统/)。

| ENT | 文化与概念实体 | [ENT-CULTURE](../真相/10-百科/文化/ENT-CULTURE-企业文化与话语口径.md) |
| NAR | 叙事合同 | [NAR-00](../真相/20-叙事合同/NAR-00-叙事母题与基调.md) |
| PCM | 主角与主线（主角视角的设计母本，非逐字剧情） | [PCM-00](../真相/30-主角与主线/PCM-00-使用范围与边界.md) |
| TIM | 时序与历史 | [TIM-00](../真相/40-时序与历史/TIM-00-时序索引.md) |
| CHR | 人物（制度锚点） | [CHR-SOLEN-ELI](../真相/10-百科/人物/CHR-SOLEN-ELI-伊莱·索伦.md) |

## 第二条　文件名

Markdown 文件名建议为：`{id}-{中文简名}.md`，便于人类浏览与工具检索并存。

## 第三条　Frontmatter 必填字段

```yaml
id: ORG-EXAMPLE
title: 示例组织
status: active | draft | deprecated
visibility: internal | partner | public
```

扩展字段允许：`aliases`、`supersedes`、`related`。

## 第四条　中文标题

`title` 要为正式中文称谓，与对外宣传用语保持可追溯关系；若对外使用英文名，在 `aliases` 中注明。

## 第五条　内部链接

正文互引须符合 [03-内部链接书写规则.md](03-内部链接书写规则.md)；合并前运行 [工具/检查/check-links.py](../工具/检查/check-links.py)。

## 第六条　CHR 条文节序（建议）

人物条文（`CHR-*`）建议采用下列节标题，与 [TIM-04](../真相/40-时序与历史/TIM-04-前任管理员执政与第九区事态背景年表.md) 等时序条文分工：**时间线**写人物经历锚点，**不**用「年表锚点」；**职权与牵制**写职务权柄与科层博弈，**不**用「制度接口」。

| 节 | 标题 |
|----|------|
| 第一条 | 登记与职务 |
| 第二条 | 玩家情报分层（B-9002） |
| 第三条 | 时间线 |
| 第四条 | 职权与牵制 |
| 第五条 | 关联条文 |

## 第七条　专名：前任管理员

第九区事变前的管理者人工智能，设定专名为 **前任管理员**（可简称 **管理者**／**管理员** 作口语，但须让人听懂指谁）。

- **禁止**单独用「前任」指代该角色（易与「前任科长」「上一任」等人事继任混淆）。
- **禁止**用「前任桥」替代专名；**桥**仅作制度类型（如 B-9002），不是该角色的名字。
- 人事继任语境可写 **继任科长**、**下一任服务科科长**、**前任服务科科长**（人类），与 **前任管理员** 分开。
- **禁止**用「接印」「接科长印」等说法。

## 第八条　设定专名：非实控区

**非实控区**指第九区等城市内**名义上仍属企业管控范围、实际管控薄弱**，帮派与外包秩序并存的地带（如「社区非实控区」「非实控区谈判」「非实控区组织」），为**已定设定专名**。

- **制度口径**：名义上**整座城市皆为管控区域**；「非实控」强调**实际执行与名义覆盖之间的落差**，不是「城外法外之地」。
- **废止旧称**：**灰区**（写作黑名单）；**非正式辖区**（过渡用语，全文改用 **非实控区**）。
- 首次在跨库文档引用时，可写「**非实控区**（社区非实控区）」以消歧。
- 工程/规范文档中的待对齐、边界不清等语义，仍用待对齐项、部分符合等具体说法（见 `.cursor/rules/no-coined-terms.mdc`，本地保留、不进 Git）。
