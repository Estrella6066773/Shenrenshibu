# 阅读起点（设定设计）

本文档概述 **设定设计/** 内的分层结构。全库入口见仓库根 [**00-文档库总览.md**](../00-文档库总览.md)。

本层同时服务于**长期协作撰稿**与**大规模玩家考据**；进入具体条文前，请先把握层级边界。

## 一、五层结构（职责不可混用）

| 层级 | 路径 | 用途 |
|------|------|------|
| 设定真值 | [真相/](真相/README.md) | 内部完整世界观与机制；写作与审核的**唯一锚点**。 |
| 主角与主线（设定真值子区） | [真相/30-主角与主线/](真相/30-主角与主线/README.md) | 主角／B-9002 视角的设计母本；与百科机制、叙事合同的分工见 [PCM-00](真相/30-主角与主线/PCM-00-使用范围与边界.md)。 |
| 叙事合同 | [真相/20-叙事合同/](真相/20-叙事合同/README.md) | 母题、基调、红线、终局分型；约束一切对外叙事口径。 |
| 叙事生产 | [叙事生产稿/](叙事生产稿/README.md) | 任务简报、脚本、关卡叙事等**可过期**生产稿。 |
| 对玩家冻结 | [玩家投放/](玩家投放/README.md) | 已批准对外披露内容，**按版本目录**归档。 |
| 治理与管线 | [管理规则/](管理规则/README.md)、[工具/](工具/README.md) | 命名、`id`、可见性、内部链接、导出与一致性检查。 |

- 每一条正式设定都配有稳定 **`id`**（见各文 frontmatter 与 [管理规则/id-registry.yaml](管理规则/id-registry.yaml)）。
