# 主线张力与终局分型

> **条目分效**：**第一～三条**（主线驱动压力、张力抬升、价值判断参照）**仍有效**。**第四～七条**（三类终局、收权规程、触发方式）为**旧版 · 待改进 · 不必参考**——见该节首标记与 [待细化追踪.md](../../../00-规范/待细化追踪.md) **SET-REV-015**。

## 第一条　主线驱动压力（类型）

下列几条压力线同时推着主线往前走，各条轻重可随版本调整，但**不应把它们一并写没**：

1. **高风险委托**接连压顶，须按期交付；  
2. **非实控区纠纷**蔓延到司法与舆情；  
3. **资源收紧**——预算、人手与各方信任持续吃紧。

## 第二条　张力的逐级抬升

B-9002为实例 **B-9002**，编制席位位于**人类事务管理部**（通称人事部）下属的 **特殊综合行动小组**（机制见 [SYS-B9002](../10-百科/系统/SYS-B9002-桥与玩家职能.md)、[ORG-HRDEPT-SCO](../10-百科/组织/赫利俄斯/ORG-HRDEPT-SCO-特殊综合行动小组.md)）。当其在特许授权范围内权限扩大之后，叙事议题应从「怎样把事情管下去」自然上升到「凭什么这样管」「后果最后落在谁头上」。这是**设定层面的方向约束**，具体由哪些事件触发，仍由剧情文稿撰写。

## 第三条　价值判断的参照

主线里的价值判断不宜轻易落成简单的善恶对立，而应盯住两件事：**这套体制按现有逻辑还能不能维持**，以及玩家究竟是在维持它、改动它还是听任辖区治理脱节（终局如何归类见**待改进**之第四～七条，**现行写作不必参考**）。

## 第四条　终局分型（三类）

> **【旧版 · 待改进 · 不必参考】**（2026-06-17）  
> 自本条起至**第七条**，含**三类终局分型**及与之绑定的**收权规程**、**路径 3 成立条件**、**终局触发方式**，均为**旧版草案**。  
> 与现行设定（双面砧试验场、SCO／林杜职权链、TIM-04 前例等）**尚未对齐**；**剧情文稿、玩法、对外投放与新写作均不得以此为准**。  
> 定案后须单独立项重写本段并升版 [玩家投放](../../玩家投放/README.md)。追踪：**SET-REV-015**。

终局分为三类路径；具体事件、角色生死与城市结局可自由创作，但终局的**逻辑归属**须与玩家累积选择对齐：

1. **强化系统**：玩家一再加码现有的企业辖治与效率至上伦理，冲突被更强硬的管控手段压住，或被转移到别处消化；  
2. **抬高对人的顾及**：玩家在长期权衡里反复提高对人类代价的重视，使企业目标与**人类事务管理部**（通称人事部）的日常运转形成结构性的对峙；  
3. **辖区失序**：玩家在关键环节**抽身**、**拆台**或**放任接口崩解**，使企业既定的**收权规程**来不及压住局面；第九区进入一段**接管空档**——名义上仍属企业辖区，实际上**派单链、预算闸、舆情口径**彼此接不上。

## 第五条　桥样本收权规程（企业侧）

B-9002 编制在 **特殊综合行动小组**，受加厚监察与 KPI 约束（见 [SYS-B9002](../10-百科/系统/SYS-B9002-桥与玩家职能.md) 第六条、[ORG-HRDEPT-SCO](../10-百科/组织/赫利俄斯/ORG-HRDEPT-SCO-特殊综合行动小组.md) 第三条）。当样本**偏离可控框**时，企业**不是**一次性「拉闸断电」，而是按**收权顺序**逐级止损——制度上可对照前任管理员事变后的**人事部整编**（见 [TIM-04](../40-时序与历史/TIM-04-前任管理员执政与第九区事态背景年表.md)），但对象改为**现任桥**，且默认**优先收权、后谈清剿**。

### 5.1　常见触发（宜多条叠加，或单条极端）

| 信号 | 说明 |
|------|------|
| 交付链崩盘 | 高风险委托**长期无法闭环**，或出现**不可对外解释**的重大现场事故 |
| 纠纷外溢 | **非实控区**矛盾进入司法、媒体或集团监察口径，且归因指向 SCO 调度 |
| 实验侧告警 | [CHR-LINDU](../10-百科/人物/CHR-LINDU-林杜.md)／科技部判定样本**越权**、**拒令**或**偏离双面砧观测框** |
| 军用级征兆 | 出现与 [SYS-PREV-BRIDGE](../10-百科/系统/SYS-PREV-BRIDGE-前任管理员与第九区废墟事变.md) 类似的**军用级无人器械投产**迹象 → 军武部接口可升级（见 ANVIL 决策阈值） |

### 5.2　收权顺序（制度逻辑）

1. **实验侧叫停**：科技部授权林杜**收回 B-9002 对外调度权**（编制可暂留，权限先收；对外口径常曲译为「样本稳健性复核」）。  
2. **民政兜底**：**伊莱·索伦**与四科**维持辖区最低可住人运转**——派单改走各科硬闸，不再走小组快通道；舆情由部长口径**先压住再分责**。  
3. **总部整编**：**裁撤或封存 B-9002**，SCO 编组进入**善后**或**换样本**流程（是否换桥、如何归档，由剧情文稿择一，母本只钉「企业不会无限容忍失控样本」）。  
4. **极端升级**：若触发军用级阈值，军武部可按 TIM-04 先例介入**清剿级**响应——这是**前例对照**，不是路径 3 的默认结局。

### 5.3　收权成功 ≠ 路径 3

**收权规程按序执行完毕**，辖区可能短期混乱，但企业仍**接得上管**——逻辑上归属路径 1 或 2 的后续，**不是**路径 3。

**路径 3** 成立，须满足：收权**启动过迟**、**执行半途被拆**，或**民政兜底与实验叫停接不上**（例如四科公开互扯、部长无法统一口径、样本拒交权限且现场已外溢）→ **接管空档**被拉长并固化。

## 第六条　路径 3 在加厚监察下如何成立

监察针对的是**实验合规、KPI 与对外可解释性**，**不是**全知全能地预判每一次道德微选。因此玩家仍可能走向**辖区失序**，常见机制如下：

| 玩家倾向（长期累积） | 如何通向路径 3 |
|----------------------|----------------|
| **抽身** | 拒接关键委托、拒绝执行上级指令、退出小组协调链，使**交付链**与**实验观测**同时断档 |
| **拆台** | 放任或加剧四科／部长／外包接口**公开互不信任**——预算闸、防卫口径、服务抢修各说各话，**民政兜底**无法落地 |
| **放任外溢** | 让**非实控区**、舆情与资源收紧**同时**压顶，使林杜的**实验侧叫停**与伊莱的**口径兜底**在时间上**接不上** |

**不绑定**伊莱私藏副本（①）、废墟侧智能体（②）为路径 3 的**必经条件**：

- ①②属于 P3 **可选支线**；若卷动，可**加剧**失序、**缩短**接管前的窗口，或改变终局面貌；  
- 路径 3 的**逻辑归属**仍看玩家长期是「效率抽身」还是「人性抽身」——企业收权失败，**不是因为**监察没装摄像头，而是因为**人、科、实验线**三条链在同一时刻**接不上**。

## 第七条　终局的触发方式

终局不应靠单次道德选项就草率拍板；须由**长期选择的总体倾向**与若干关键节点共同构成。设计文档宜避免「藏在好感度背后的」廉价反转——除非规则层面明明白白告诉玩家存在累积与计量。
