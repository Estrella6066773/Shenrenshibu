# 实现：存档与进度（SaveProgress）

| 字段 | 值 |
|------|-----|
| 模块键 | SaveProgress |
| 主代码路径 | `Assets/01_Scripts/Runtime/Save/`、`Assignment/Controller/Modules/SaveProgressBackendAdapter.cs` |
| 程序集 | `ShenRenShibu.Runtime`、`ShenRenShibu.Domain` |

## 1. 职责边界

- **负责**：存档集与子存档索引、主存档指针、编排层 API、旧单文件迁移。
- **不负责**：各模块字段序列化细节（仍由 `AssignmentSaveReplayService` 承担）。

## 2. 存档集模型

| 术语 | 说明 |
|------|------|
| 存档集 | 列表一块；上限 100 |
| 子存档 | 每次手动/自动保存追加；历史保留 |
| 主存档 | `primarySnapshotId`；继续游戏读此档 |

磁盘：`persistentDataPath/campaign_saves/`

- `collection_index.json`
- `collections/{id}/collection.meta.json`
- `collections/{id}/snapshots/{snapId}.json`

## 3. 核心类型

| 类型 | 说明 |
|------|------|
| `CampaignSaveCollectionService` | 集/子档 CRUD、排序（标星优先） |
| `SaveProgressOrchestrator` | 对外编排 API |
| `SaveProgressBackendAdapter` | Assignment 后端适配（JSON 桥接，避免 Runtime→Domain 类型泄漏） |
| `CampaignAutoSaveTrigger` | 日结自动保存（Domain/PlayerState） |

## 4. UI 入口

- 主菜单 `SaveManagementPanelController` + 预制体 `UI_存档管理面板`
- 游戏内底部导航：快速存档写活动集；管理存档回主菜单
