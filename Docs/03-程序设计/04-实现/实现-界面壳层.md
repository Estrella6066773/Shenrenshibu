# 实现：界面壳层（UI_Shell）

| 字段 | 值 |
|------|-----|
| 模块键 | UI_Shell |
| 主代码路径 | `Assets/01_Scripts/Runtime/Shell/`、`Presentation/` |
| 程序集 | `ShenRenShibu.Runtime`、`ShenRenShibu.Presentation` |

## 1. 职责边界

- **负责**：玩法事件弹窗排队、对话播放中抑制/延后、同类型弹窗不叠层。
- **不负责**：邮件/聊天 UI 布局（MessageSystem）；委托详情面板展开队列（Assignment View）。

## 2. 入口

| 类型 | 说明 |
|------|------|
| `GameplayPopupCoordinator` | 运行时弹窗优先级协调（Runtime） |
| `AssignmentInteractionUIManager` | 委托弹窗生产者，经 Coordinator 展示 |
| `DialogPlaybackController` | 对话播放时设置 `DialogPlaying` 状态 |

## 3. 规则（REQ-UI-001~002）

1. 委托阶段/过期/失败结算弹窗经 Coordinator 单队列展示。
2. 对话播放中，新玩法弹窗入队，对话结束后继续。
3. 关闭弹窗后恢复底层交互。
