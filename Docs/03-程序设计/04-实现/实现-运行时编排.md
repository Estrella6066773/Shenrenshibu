# 实现：运行时编排（RuntimeOrchestration）

| 字段 | 值 |
|------|-----|
| 模块键 | RuntimeOrchestration |
| 主代码路径 | `Assets/01_Scripts/Runtime/` |
| 程序集 | `ShenRenShibu.Runtime` |

## 1. 职责边界

- **负责**：跨模块执行顺序、启动门控发布、日结步骤编排、危机/叙事/存档编排入口。
- **不负责**：各玩法模块内部规则（委托分支、压力数值、矛盾结算等）。

## 2. 场景挂载

| 组件 | 路径 | 顺序 |
|------|------|------|
| `EventBusBootstrap` | 场景根 | -2000 |
| `RuntimeBootstrap` | 场景根 | -1900 |
| `StartupGateCoordinator` | Runtime 根 | -1890 |
| `DayCyclePipeline` | Runtime 根 | -1880 |
| `CampaignSaveOrchestrator` | Runtime 根 | -1870 |
| `GameplayPopupCoordinator` | Runtime 根 | -1860 |
| `StoryProgressService` | Runtime/Progress | -1850 |
| `DepartmentUnlockRegistry` | Runtime/Progress | -1840 |

Editor 菜单：`神人事部/场景/安装运行时编排根`。

## 3. 核心类型

| 类型 | 文件 | 说明 |
|------|------|------|
| `RuntimeBootstrap` | `Bootstrap/RuntimeBootstrap.cs` | 安装并串联编排器 |
| `StartupGateCoordinator` | `Gate/StartupGateCoordinator.cs` | 发布 `RuntimeInitializationCompletedEvent` |
| `DayCyclePipeline` | `DayCycle/DayCyclePipeline.cs` | 订阅 `OnDayChanged`，按序执行 `IDayCycleStep` |
| `IDayCycleStep` | `Abstractions/IDayCycleStep.cs` | 日结步骤接口 |
| `CampaignSaveOrchestrator` | `Save/CampaignSaveOrchestrator.cs` | 战役存档读写编排 |
| `ICampaignSaveBackend` | `Abstractions/ICampaignSaveBackend.cs` | 存档后端（由 Assignment 适配） |
| `GameplayPopupCoordinator` | `Shell/GameplayPopupCoordinator.cs` | 玩法弹窗与对话播放优先级 |
| `CrisisEventCoordinator` | `Crisis/CrisisEventCoordinator.cs` | 压力/矛盾事件选取顺序（stub） |
| `NarrativeDispatchCoordinator` | `Narrative/NarrativeDispatchCoordinator.cs` | 阶段 hook 路由登记（stub） |
| `AssignmentTickHost` | `Assignment/AssignmentTickHost.cs` | 委托 Tick 统一入口（stub） |

## 4. 与各模块集成

| 模块 | 集成方式 |
|------|----------|
| PlayerEconomy | `OnDayChanged` → `DayCyclePipeline.Run` |
| Assignment | `AssignmentDailyRefreshDayStep`、`CampaignSaveBackendAdapter` |
| DialogSystem | `DialogPlaybackController` 通知 `GameplayPopupCoordinator` |
| UI_Navigation | 存读档按钮 → `CampaignSaveOrchestrator` |
