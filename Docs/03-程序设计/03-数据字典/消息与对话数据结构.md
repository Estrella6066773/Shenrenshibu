# 消息与对话数据结构

## 主要类型

| 类型 | 归属 | 说明 |
|------|------|------|
| `ChatChannelRegistry*` | SO 配置 | 聊天频道注册表与排序来源 |
| `ChatRoleDatabase*` | SO 配置 | 角色头像、名称与发言身份 |
| `ChatStorySO` | SO 配置 | 聊天剧情配置 |
| `MailEventDatabaseSO` | SO 配置 | 邮件事件索引 |
| `DialogConfig` / `DialogNodeConfig` | 运行时配置 | 对话节点播放与选项 |
| `ChatHistoryStorageService` | 运行时服务 | 聊天历史与已读状态 |

## 消息系统 SO

| 数据 | 规则 |
|------|------|
| 频道 ID | 稳定唯一，聊天记录与剧情配置引用该 ID |
| 角色 ID | 稳定唯一，与设定层角色名可关联但不直接依赖显示名 |
| 邮件事件 ID | 压力、矛盾、委托桥接使用的投递键 |
| 排序字段 | 会话列表排序依据，置顶、未读、时间、配置顺序须可追溯 |
| 展示文本 | 可引用剧情文稿，但运行时代码不写死正文 |

## 对话节点

| 字段 | 规则 |
|------|------|
| 节点 ID | 同一对话配置内唯一 |
| 说话角色 | 引用角色库或运行时角色键 |
| 文本内容 | 来自配置，程序只负责播放 |
| 选项 | 选项节点可暂停播放并等待玩家选择 |
| 后继节点 | 线性或分支后继，必须能被存档状态还原 |

## 运行时状态

| 状态 | 说明 |
|------|------|
| 已见消息键 | 防止重复刷出已播放消息 |
| 已解决选项节点 | 防止重复弹出已处理选项 |
| 选项选择记录 | 记录玩家选过的回复，供后续节点读取 |
