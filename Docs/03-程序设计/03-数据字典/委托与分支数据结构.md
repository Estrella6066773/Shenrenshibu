# 委托与分支数据结构

## 主要类型

| 类型 | 归属 | 说明 |
|------|------|------|
| `AssignmentTemplateSO` | SO 配置 | 委托模板、默认参数与任务展示信息 |
| `AssignmentBranchTreeSO` | SO 配置 | 分支树、规则条件与分支结果 |
| `AssignmentBranchTreeDatabaseSO` | SO 配置 | 分支树索引 |
| `AssignmentData` | 运行时模型 | 已创建委托实例状态 |
| `AssignmentBelongingKind` | 枚举 | `Daily`、`StorySeries`、`Event`、`Other` |

## `AssignmentTemplateSO`

| 字段 | 规则 |
|------|------|
| 委托 ID | 稳定唯一；存档、邮件与剧情引用不得使用显示名 |
| 显示名与描述 | 面向玩家展示，可引用设定层文本但不写剧透层信息 |
| 类型归属 | 使用 `AssignmentBelongingKind`，日常委托必须标记为 `Daily` |
| 默认时长与奖励 | 策划参数走 SO，不在 Tick 代码中硬编码 |
| 分支树引用 | 指向 `AssignmentBranchTreeSO` 或数据库条目 |
| 任务效果 | 只记录配置入口，执行由 `TaskInteractionEffectEngine` 处理 |

## `AssignmentBranchTreeSO`

| 字段 | 规则 |
|------|------|
| 节点 ID | 同一分支树内唯一 |
| 前置条件 | 由规则解析器解释，不在 UI 层判断 |
| 权重或优先级 | 若参与随机或排序，必须可配置 |
| 进入结果 | 写明任务状态、消息 hook、压力或矛盾桥接事件 |
| 失败处理 | 明确无可用分支时的回退行为 |

## `AssignmentData`

| 字段 | 规则 |
|------|------|
| 实例 ID | 运行时唯一，存档稳定 |
| 模板 ID | 指向创建来源 |
| 当前进度 | Tick 更新，结算须幂等 |
| 员工槽位 | 保存员工引用，不复制员工内部状态 |
| 当前分支 | 保存节点或阶段标识，便于存档回放 |

## 日常委托池

日常刷新只读取 `AssignmentBelongingKind = Daily` 的模板。场景可配置 `fallbackDailyTemplateIds`，但 fallback 仍应指向正式模板，不应在代码中写死完整委托内容。
