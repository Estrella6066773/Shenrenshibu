# 员工与标签数据结构

## 主要类型

| 类型 | 归属 | 说明 |
|------|------|------|
| `EmployeeData` | 运行时模型 | 员工基础身份、状态、标签列表与委托关联 |
| `EmployeeTagSO` | SO 配置 | 单个员工标签定义 |
| `EmployeeTagDatabaseSO` | SO 配置 | 标签库与编辑器创建入口 |
| `TagBuffSnapshot` | 运行时快照 | 标签 Buff 聚合后的跨模块读取结果 |
| `TagModifierChannel` | 枚举 | 日薪、个人效率、压力获得等修正通道 |

## `EmployeeTagSO`

| 字段 | 类型 | 规则 |
|------|------|------|
| 标签标识 | 字符串或派生类型字段 | 同一标签库内唯一 |
| 显示名 | 文本 | 面向玩家展示，须使用词条库认可用语 |
| 分类 | 枚举或子类 | 性格、义体、状态等分类，不在 UI 层临时判断 |
| Buff 条目 | 数组或列表 | 每项写明通道、运算方式、数值与说明 |
| 说明文本 | 文本 | 仅做展示，不参与数值结算 |

## `EmployeeData`

| 字段 | 规则 |
|------|------|
| 员工 ID | 运行时与存档稳定引用，不使用显示名做主键 |
| 状态 | 由 Employee 模块公开入口变更 |
| 标签列表 | 只保存引用或稳定标识，Buff 由聚合器即时解析 |
| 当前委托 | 与 Assignment 模块协作，不直接写委托内部状态 |

## `TagBuffSnapshot`

| 字段 | 规则 |
|------|------|
| 通道值 | 按 `TagModifierChannel` 分组 |
| 来源明细 | 保留贡献来源，供 UI 与调试展示 |
| 聚合结果 | 消费模块只读，不回写标签配置 |

## 示例关系

```mermaid
flowchart LR
  TagDatabase[EmployeeTagDatabaseSO]
  TagSO[EmployeeTagSO]
  Employee[EmployeeData]
  Snapshot[TagBuffSnapshot]
  TagDatabase --> TagSO
  TagSO --> Employee
  Employee --> Snapshot
```
