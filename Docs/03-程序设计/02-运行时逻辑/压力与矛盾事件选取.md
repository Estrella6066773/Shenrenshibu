# 压力与矛盾事件选取

本文记录压力事件与矛盾事件在运行时的选取顺序与去重占位。规则收束见 [需求-矛盾与压力事件选取](../../../02-系统设计/02-需求/需求-矛盾与压力事件选取.md)。

## 目标顺序（流程设计详情图）

```mermaid
flowchart TD
  tick[委托 Tick 桥接]
  stress[压力增量 ApplyStressDelta]
  tier[分段跨越通知]
  pickStress[达阈值选压力模板]
  inject[可向任务注入矛盾]
  conflict[任务内矛盾累积]
  pickConflict[达阈值选矛盾模板]
  mail[邮件桥接投递]
  tick --> stress --> tier --> pickStress
  pickStress --> inject --> conflict --> pickConflict --> mail
```

## 代码落点

| 职责 | 当前 | 目标 |
|------|------|------|
| 选取顺序编排 | 模块内隐式链 | `Runtime/Crisis/CrisisEventCoordinator`（stub） |
| 压力规则 | `StressSystem` | 不变 |
| 矛盾规则 | `ConflictSystemService` | 不变 |
