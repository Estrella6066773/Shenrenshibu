# 任务内交互效果

本文记录委托任务内的条件效果与跨员工相互作用。该文档自 `Assets/01_Scripts/Assignment/Docs/` 迁入，模块目录内不再维护 Markdown 文档。

## 目标

在不破坏员工标签常驻效果的前提下，引入「任务内条件生效 + 跨员工相互作用」能力。当前版本可按 Tick 汇总上下文，并向成员下发效率倍率与能力百分比修正。

## 兼容原则

- `EmployeeTagSO.buffModifiers` 继续作为常驻层。
- 任务内交互效果作为动态层，只在具体委托任务内生效。
- 若动态层没有注入结果，旧公式行为保持不变。

## 主要类型

| 类型 | 说明 |
|------|------|
| `TaskTeamContext` | 某一帧任务小队的只读快照，提供成员、职位需求、任务状态等条件判断输入 |
| `TaskEffectResult` | 一次求解的输出结果，以 `employeeId` 为粒度保存运行时修正 |
| `IConditionalRuleSolver` | 复杂规则插件接口，例如傲慢、嫉妒等标签规则 |
| `TaskInteractionEffectEngine` | 规则调度器，负责注册与执行 solver |
| `RuntimeSettlementChannel` | 统一结算通道，便于压力、矛盾等系统后续复用 |
| `IAssignmentRuntimeSettlementView` | 运行时结算视图接口 |

## 当前实现范围

已实现：

- 类型定义与运行时容器。
- 引擎基础调度与 solver 注册。
- 结果容器的默认初始化。
- `AssignmentManager` 每 Tick 汇总任务成员上下文并执行引擎。
- `AssignmentController` 支持注入成员效率倍率并参与进度计算。
- 内置规则最小实现：`ArrogantRuleSolver`、`JealousRuleSolver`。
- `AssignmentManager` 暴露规则参数，可在 Inspector 调整标签 ID、别名、阈值与百分比。
- `EmployeeTagSO` 支持 `taskInteractionRules`，可按标签声明 `solverKey` 与参数。
- `AssignmentManager` 可通过 `taskInteractionTagDatabase` 读取标签规则，并按成员聚合。
- `AssignmentController` 支持按 `sourceId` 注入并聚合：效率与通道倍率按来源连乘，能力百分比按来源求和。
- 运行时员工数据查询改为 `EmployeeRuntimeDataRegistry`，不再直接依赖员工卡牌视图。
