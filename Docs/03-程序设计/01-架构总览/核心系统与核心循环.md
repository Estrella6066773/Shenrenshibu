## 玩家主循环（摘要）

> 说明：`STR 压力`绑定员工，`CNF 矛盾`绑定任务。员工压力事件可能向任务注入矛盾，任务内矛盾事件则需玩家协调解决。

```mermaid
flowchart LR
  subgraph 人事
    RSM[简历 / 招聘]
    EMP[员工与标签]
  end
  subgraph 运转
    ASG[委托 / 任务]
    PES[日结 / 经济]
  end
  subgraph 张力
    STR[员工压力]
    CNF[任务矛盾]
  end
  subgraph 终端
    MSG[邮件 / 聊天]
    DLG[剧情对话]
  end
  RSM --> EMP
  EMP --> ASG
  ASG --> PES
  ASG --> STR
  STR --> CNF
  PES --> CNF
  MSG --> DLG
  ASG --> MSG
```

## 启动与门控

- `EventBusBootstrap` 安装总线后，由 `StartupGateCoordinator` 发布 `RuntimeInitializationCompletedEvent`；委托等系统经门控放行（详见 `实现-核心消息与启动门控`、`实现-运行时编排`）。
- 测试场景仍可使用 `Tests/Runtime` 下 Bootstrap 覆盖门控。

## 数据与表现

| 层 | 位置 |

## 设定库边界

- **机制设定**（委托优先序、冲突规则等）：StoryLib `真相/10-百科/规则/`。  
- **任务简报与节拍**：`剧情文稿/`、`30-主角与主线/`。  
- 程序实现不得与「设定母本」冲突；剧情文稿冲突时以母本为准。
