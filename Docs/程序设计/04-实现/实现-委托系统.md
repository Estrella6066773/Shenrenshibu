# 实现：委托系统（Assignment）

| 字段 | 值 |
|------|-----|
| 模块键 | Assignment |
| 关联设计 | `需求-委托系统.md`、`需求-日常委托刷新.md`、`需求-地图与任务布点.md`、`需求-界面与弹窗.md` |
| 主代码路径 | `Assets/01_Scripts/Assignment/` |

## 1. 职责边界

- **负责**：管理器命令处理、实例工厂、运行时 Behaviour + 纯逻辑 `AssignmentController`、分支树、任务内效果引擎、任务消息枢纽、存档回放与就绪校验等。
- **不负责**：全局总线安装（Core）；员工模型定义（Employee）。

## 2. 入口与顺序

- `AssignmentManager`（`DefaultExecutionOrder(-1500)`）：`Modules/AssignmentManager.cs`  
  - `requireInitializationGateMessage` / `requiredGateKey` 与 `RuntimeInitializationCompletedEvent` 对齐。  
  - `Awake` 内构造 `TaskInteractionEffectEngine` 并注册内置 `*RuleSolver`。
- `AssignmentControllerBehaviour` + `AssignmentController`：`Tick` 推进（详见架构图文档第四节）。
- 本地消息：`Assignment/Messaging/AssignmentTaskMessageHub.cs`（与全局总线分离）。

## 3. 关键数据与配置

- `AssignmentData`、`AssignmentFlowModels`
- `AssignmentTemplateSO`、`AssignmentBranchTreeSO`、`AssignmentBranchTreeDatabaseSO`
- 结构说明（若存在）：`Assignment/委托对象结构.md`

## 4. 集成

- 阶段 hook：`PhaseIntegrationHookRequested` 仅传 `hookId`，由 UI/叙事消费。
- 员工效率与矛盾/压力：`EmployeeWorkStateResolver`、各 Bridge；进度计算见架构图。

## 5. 日常池与类型（REQ-ASG-007~011）

- `AssignmentBelongingKind`：`Daily`、`StorySeries`、`Event`、`Other`（见 `AssignmentData.cs`）。
- **周期发布（REQ-ASG-011）**：`AssignmentDailyRefreshPublisher` 订阅 `PlayerEconomySystem.OnDayChanged`，在日结跨天后从 `AssignmentPrefabRegistry` 中 `belongingKind=Daily` 的模板随机发布（默认 **3** 条/日）。
- 场景须挂载 `AssignmentDailyRefreshPublisher` 并绑定经济与注册表；注册表条目需配置 `templateOverride` 且 `belongingKind=Daily`，或填写 `fallbackDailyTemplateIds`。
- 策划默认见 [需求-日常委托刷新](../../../系统设计/02-需求/需求-日常委托刷新.md) §4。
