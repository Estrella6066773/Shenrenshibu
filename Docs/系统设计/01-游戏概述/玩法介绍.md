# 玩法介绍

> 面向玩家与外部协作者的玩法说明，整理自 `02-需求` 与 `01-架构总览`。
>
> 最后更新：2026-06-21

## 一句话定位

《神人事部》是一款以**人事调度与危机管理**为核心的模拟经营游戏。玩家扮演“人类事务管理部（简称人事部）”的管理者，在有限的时间与资金约束下，招聘员工、分配委托、处理矛盾事件与员工压力事件，并通过邮件、聊天与剧情对话推进城市中的故事。

## 玩家扮演谁

你是人事部的一名管理者。手下有一群能力、性格、标签各异的员工；城市中不断涌现出各种委托请求；而任务推进过程中，员工与员工之间、员工与任务环境之间的矛盾会逐渐累积，最终演变成需要你来协调的事件。你必须在**效率、成本、人情与压力**之间寻找平衡。

## 核心循环

游戏的主循环围绕「人」与「事」两条线展开：

1. **招募、管理员工**：浏览简历，招聘合适的员工；查看员工能力、状态与标签 Buff；把员工安排到合适的位置。
2. **接取委托**：每游戏日会从已解锁的任务库中刷新若干日常委托，玩家可以挑选接取；此外，也存在用于推进剧情的委托，会在符合条件后自动生成在地图上。
3. **分配与执行**：把员工派进委托的不同职位，任务会随时间自动推进。
4. **日结与结算**：每过一天（默认 60 秒现实时间 = 1 游戏日）触发日结，扣除员工薪资，并根据任务完成或失败结果进行资金结算。
5. **压力与矛盾**：员工在执行委托时会积累**压力**；员工压力过大时会触发**压力事件**——一类绑定到具体员工的突发危机，可能带来邮件通知、针对所在任务的临时效果（如效率变化、职位限制）、员工状态变化等效果。**矛盾**则在任务域内不断累积；当任务内矛盾达到阈值后生成**矛盾事件**——一类绑定到具体任务的案件，需要玩家亲自协调涉事员工并作出结算。
6. **消息与剧情**：委托阶段、矛盾事件、剧情进度等会通过邮件、聊天和剧情对话投递给玩家，驱动叙事与世界观展开。

## 主要玩法系统

### 1. 员工管理

- 每位员工拥有能力值、状态与一组标签。
- 标签可以叠加 Buff，影响员工的能力、日薪、工作效率、压力获取等多个通道。
- 员工状态包括闲置、委托中、待处理、矛盾中、离职等。
- 你可以分配、解除分配、开除员工；开除时需要支付赔偿金。

### 2. 委托系统

- 委托分为日常、系列/常驻、剧情等类型。
- 每游戏日日结后，日常委托池会刷新 3 条新委托（首版默认值）。
- 玩家可以开始系列/常驻类任务，也可以在任务进行中随时叫停——叫停视为失败，会按失败规则结算奖励与压力。
- 每个委托由多个阶段组成，阶段推进会触发进度、聊天、邮件等事件。
- 任务失败有重试机制，但单环节重试次数超过上限后会按失败处理。

### 3. 压力系统

- 每位员工拥有独立的压力值，所有压力增量都会聚合到对应员工上。
- 员工压力分为多个分段，跨越分段时会对外通知，UI 与其他系统可据此响应。
- 员工压力过高会触发**压力事件**——这是压力系统的专有危机单元，绑定到具体员工，按员工性格标签匹配模板，可能附带邮件通知、针对所在任务的临时效果（如效率变化、职位限制）、员工状态变化等效果；员工压力达到上限时还会触发更剧烈的「满值性格失控」事件。
- 压力事件有去重机制，避免短时间内反复触发。
- 某些压力效果只作用于特定委托，任务结束或过期后自动消失。

### 4. 矛盾系统

- **矛盾**是任务域内的一种累积资源，绑定到具体任务，由压力、员工标签、任务推进等因素共同驱动。
- 任务内矛盾达到阈值后，从模板池中生成具体的**矛盾事件**——这是矛盾系统的专有案件单元，绑定到具体任务，需要玩家亲自介入协调。
- 玩家对矛盾事件进行协调结算；结算逻辑保证幂等，避免重复扣款或发奖。
- 涉事员工会进入「矛盾中」状态，与委托产出规则对齐。
- 矛盾事件支持冷却、去重与日终超时。

### 5. 消息终端

- 游戏内提供邮件、聊天、日志等终端界面。
- 邮件由策划配置的邮件库驱动，支持列表、详情与结果页。
- 聊天支持频道管理、历史记录存档，列表按置顶、未读数、最近消息时间排序。
- 剧情对话按节点线性播放，自动逐条出消息，遇到选项时暂停，等待玩家选择后继续。

### 6. 简历与待命区

- 简历窗口提供列表、详情、拖拽排序与选中高亮等交互。
- 待命区是身份牌的兜底收纳区：无法放置的员工牌会自动回收至此，也支持一键整理布局。

### 7. 地图与任务标记

- 世界由一组预置地点构成，委托创建时可绑定地点，未绑定则随机选取。
- 进行中的委托会在地图上显示任务标记，点击标记或任务方块可打开任务面板。
- 地图不是开放世界，而是服务于任务布点与快速定位的信息界面。

### 8. 底部导航与界面壳层

- 游戏首版采用「主场景 + 游戏内导航」结构，没有独立的 Title 场景。
- 底部 Win 按钮可展开四主按钮，提供存档、设置、关机等子页面切换。
- 事件弹窗会按类型排队，避免同类型弹窗堆叠；剧情对话播放时，玩法弹窗会排队或被抑制。

## 叙事与世界观

游戏的故事通过以下渠道呈现：

- **邮件**：系统通知、剧情事件、压力事件与矛盾事件的后续。
- **聊天**：角色对话、群聊频道、剧情 ChatStory。
- **剧情对话**：带选项的线性对话节点，玩家的选择会影响后续走向与记忆。

世界观设定、主角主线与剧情文稿分别存放在设定母本、剧情文稿与玩家投放层中；玩法系统实现不得与设定母本冲突。
