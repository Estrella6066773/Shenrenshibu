## 1. 背景与目标

- **问题**：思维导图 B7、D14、G1 描述地图与任务地点；工程已有世界空间任务方块与点击交互，但缺**地点目录、随机选取、X 标记**的需求契约。
- **目标**：定义预置地点集合、任务落点规则、地图 UI 标记与交互验收口径（非开放世界大地图）。

## 2. 范围

- **纳入范围**：地点 ID 表、与委托绑定的世界坐标/预制引用、地图任务按钮（X 标记）展示规则、地点点击开面板。
- **不纳入范围**：街区分层设定母本（见 `设定设计/真相/10-百科/地点/`）；动态地形生成。

## 3. 功能需求

| ID | 描述 | 优先级 | 验收要点 |
|----|------|--------|----------|
| REQ-MAP-001 | 维护**预置地点集合**：每条含 `location_id`、显示名、世界坐标或场景引用 | P1 | 与 `04_Data` 或场景配置可对照 |
| REQ-MAP-002 | 委托创建时可绑定 `location_id`；未绑定则按 REQ-ASG-012 随机选取 | P2 | 读档后地点一致 |
| REQ-MAP-003 | 地图/场景内对**进行中**委托显示任务标记（导图「X 标记」；具体美术由场景配置） | P2 | 至少 1 个地点可见标记；隐藏规则与委托终态一致 |
| REQ-MAP-004 | 点击任务方块/标记打开任务面板（与 `AssignmentTaskBlockInteraction` 一致） | P1 | 冒烟：点击→面板→可操作 |
| REQ-MAP-005 | 地点集合与设定街区**可对表**但不强制 1:1（玩法地点可少于百科条目） | P3 | 文档化映射表或备注 |

## 4. 非功能需求

- 地点数量：首版建议 ≤ 20 个可复用落点（策划可调）。
- 性能：同屏标记数由场景分区控制，非本 REQ 性能专篇。
