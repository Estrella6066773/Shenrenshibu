## 1. 背景与目标

- **问题**：思维导图 G3/G4 要求二级菜单返回逻辑与事件弹窗；实现分散在底部导航、消息 Tab、委托交互 UI，缺统一 REQ。
- **目标**：定义返回栈语义与玩法事件弹窗的展示、关闭与优先级规则。

## 2. 范围

- **In Scope**：Win 子页、消息终端 Tab、委托任务面板的返回；委托阶段/矛盾/压力等事件弹窗。
- **Out of Scope**：系统级模态确认框（编辑器工具见 `no-editor-modal-dialogs` 规则）；全屏过场。

## 3. 功能需求

| ID | 描述 | 优先级 | 验收要点 |
|----|------|--------|----------|
| REQ-UI-001 | **事件弹窗**：委托阶段 hook、矛盾/压力触发等可通过专用弹窗视图展示；同屏不叠多个同类型弹窗 | P1 | 与 `AssignmentEventPopupView` 行为一致；关闭后恢复底层交互 |
| REQ-UI-002 | **弹窗优先级**：剧情对话播放中，玩法弹窗须排队或抑制（策略以实现为准，但须文档化） | P2 | 至少 1 条：对话进行中触发委托事件不崩溃 |
| REQ-UI-003 | **占位提示**：未绑定页面根时，导航按钮不抛异常（引用 REQ-NAV-003） | P1 | 开发中期场景可跑 |
