## 1. 背景与目标

- **问题**：思维导图要求独立主界面（开始游戏、继续游戏、设置）；当前工程以**游戏内**底部导航为主入口，缺 Title 场景与一键继续流程的需求锚点。
- **目标**：明确产品是否保留独立主菜单，或采纳「主场景直接 Play + 游戏内导航」策略，并写清验收口径。

## 2. 范围

- **In Scope**：开始游戏、继续游戏、设置、退出游戏的入口定义与场景流。
- **Out Scope**：多存档云同步、账号系统。

## 3. 功能需求

| ID | 描述 | 优先级 | 验收要点 |
|----|------|--------|----------|
| REQ-MENU-001 | **开始游戏**：新战役或选槽开局的路径（Title 场景或主场景等价流程） | P1 | 玩家从零可进入可玩状态 |
| REQ-MENU-002 | **继续游戏**：一键读取**最近**有效存档并恢复 | P1 | 少于 3 步到达游戏内场景；与 REQ-SAV 编排一致 |
| REQ-MENU-003 | **设置**：音量、分辨率等范围（可委托 REQ-NAV 子页） | P2 | 与 `实现-底部导航` 设置子页或 Title 设置页一致 |
| REQ-MENU-004 | **退出游戏** | P0 | 已有 REQ-NAV-002；本 REQ 引用不重复 |

## 4. 产品决策

| 选项 | 说明 | 状态 |
|------|------|------|
| A. 独立 Title 场景 | 符合白板导图；需新建场景与 UI | 备选升级 |
| B. 主场景 + 游戏内导航 | 主场景直接 Play；**开始/继续/设置/存档**由底部导航与子页承担 | **首版定案（2026-06-13）** |

**验收口径（选项 B）**：

- **REQ-MENU-001**：「开始游戏」= 新战役从默认槽或选槽进入主场景可玩态（无独立 Title 场景）。
- **REQ-MENU-002**：「继续游戏」= 底部导航「管理存档」选最近有效档，或少于 3 步恢复（与 REQ-SAV-002 一致）。
- **REQ-MENU-003**：设置范围可先与 `实现-底部导航` 设置子页对齐；独立 Title 设置页不作为首版阻塞项。
