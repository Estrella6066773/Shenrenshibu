## 1. 目标

提供终端侧 **邮件列表与详情**、**聊天 UI**、**日志** 等视图管理；支撑 **ScriptableObject 配置**（邮件库、频道注册、角色库、剧情 `ChatStory` 等）。邮件文本与聊天文本是剧情投放的核心载体；系统提供**聊天记录/频道存档** 持久化服务，并与 **剧情播放桥接**（`ChatStoryPlayer` / `ChatStoryChatBridge`）。

## 2. 功能需求

| ID | 描述 | 优先级 | 验收要点 |
|----|------|--------|----------|
| REQ-MSG-001 | `MessageUIManager` / `MessageTabUIManager` 等负责 Tab 与页面显隐，不承载核心业务规则 | P0 | UI 与数据服务分层清晰 |
| REQ-MSG-002 | 邮件数据来自 `MailEventDatabaseSO` 等配置，支持列表/详情/结果页流程 | P0 | 与策划配置的 id 一致 |
| REQ-MSG-003 | `ChatHistoryStorageService`、`ChatChannelArchiveService` 等可恢复会话与频道状态 | P1 | 重进游戏后历史一致 |
| REQ-MSG-004 | `MailDispatchService` / `MailRuntimeInbox` 运行时收件与存档模型一致 | P1 | 发件、已读状态可测 |
| REQ-MSG-005 | **聊天列表排序**：群聊/频道列表按（1）置顶标记优先（2）未读条数降序（3）最近消息时间降序；同优先级保持配置顺序稳定 | P2 | 与 `ChatUIManager` 列表行为一致；读档后排序键不丢失 |
