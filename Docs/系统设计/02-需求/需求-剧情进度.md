## 1. 背景与目标

- **问题**：`REQ-DLG-002` 仅覆盖会话记忆；思维导图 B3「剧情进度」缺**全局**进度字段与解锁 gates 的 REQ。
- **目标**：定义战役级剧情进度标识、存档字段、与日常池/科室解锁/主线节拍的对接方式。

## 2. 范围

- **In Scope**：`story_progress`（或等价）字段语义、读写模块、与存档编排（REQ-SAV）的归属、解锁 flags 引用方式。
- **Out of Scope**：具体主线文案（见 `设定设计/真相/30-主角与主线/`、`叙事生产稿/03-节拍/`）。

## 3. 功能需求

| ID | 描述 | 优先级 | 验收要点 |
|----|------|--------|----------|
| REQ-PRG-001 | 提供**战役级剧情进度**字段：可序列化、读档恢复、可被委托/消息/对话查询 | P0 | 字段列入 REQ-SAV-001 钩子清单 |
| REQ-PRG-002 | 进度由**权威模块**写入（建议：对话完成节点 + 关键消息已读 + 主线节拍触发器）；禁止多模块无序覆写 | P1 | 文档化写入点清单 |
| REQ-PRG-003 | 进度值可驱动**解锁 gates**（任务库、科室协助、底部导航页等） | P1 | 至少 1 条：进度→日常池或 UNL flag |
| REQ-PRG-004 | 与 [PCM-01 开局主线](../../设定设计/真相/30-主角与主线/PCM-01-开局主线剧情流程.md) 教程节点可对表 | P2 | 生产稿节拍与进度枚举映射表（策划维护） |
