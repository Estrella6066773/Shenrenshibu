## 1. 背景与目标

- **问题**：思维导图 D10「日常周期发布 / 任务库」标注策划待总结；当前 `需求-委托系统` 覆盖创建、Tick、结算，**未**定义日常池刷新与常驻委托限制。
- **目标**：为日常委托类型、周期发布、叫停失败、地点随机等提供可验收 REQ 锚点，供策划定稿后开发对齐。

## 2. 范围

- **In Scope**：委托类型划分（日常 / 常驻 / 剧情）、周期刷新规则占位、叫停=失败、环节重试上限、阶段 hook 与消息联动验收口径。
- **Out of Scope**：具体任务文案、数值表（由数据资产与策划表维护）。

## 3. 功能需求

| ID | 描述 | 优先级 | 验收要点 |
|----|------|--------|----------|
| REQ-ASG-007 | **委托类型**：`AssignmentBelongingKind` = Daily / StorySeries / Event / Other；类型决定 UI 文案与周期池参与规则 | P1 | 与 `AssignmentData.belongingKind`、模板 SO 一致 |
| REQ-ASG-008 | **开始任务**仅对 **StorySeries**（系列/常驻类）开放；Daily/Event 走周期池或剧情 gates | P1 | 与 `AssignmentTaskPanelDisplayText` 分类展示一致 |
| REQ-ASG-009 | **叫停任务**视为失败终态，与取消代码路径一致 | P0 | 叫停后状态=失败；奖励/压力按失败结算 |
| REQ-ASG-010 | **环节重试**：单环节最大重试次数可配置；超限按失败或跳过后续（策划二选一，默认失败） | P1 | 重试计数持久化；读档后一致 |
| REQ-ASG-011 | **日常周期发布**：每游戏日（或策划定义周期）从**已解锁任务库**检索并发布 N 条日常委托 | P0 | 周期边界触发；池为空时有明确 UX（无新日常 / 占位提示） |
| REQ-ASG-012 | **随机地点**：从预置地点集合中选取任务落点（非开放世界生成） | P2 | 与 [需求-地图与任务布点](需求-地图与任务布点.md) 地点表一致 |
| REQ-ASG-013 | **阶段 hook**：委托阶段切换时可触发聊天/邮件（契约见 MSG/DLG） | P1 | 至少 1 条冒烟用例：阶段→消息送达 |
| REQ-ASG-014 | **任务面板实时信息**：进行中委托的关键进度字段在面板可见刷新 | P2 | 与 `AssignmentTaskBlockInteraction` 面板字段对齐 |

## 4. 首版定案（REQ-ASG-011，2026-06-13）

| 项 | 定案 | 实现 |
|----|------|------|
| 周期边界 | **游戏内日结**（`PlayerEconomySystem.OnDayChanged`），与自然日解耦 | `AssignmentDailyRefreshPublisher` |
| 日常池容量 N | **3** 条/周期 | Inspector `dailyPublishCount`（默认 3） |
| 解锁任务库 | 首版池=`AssignmentPrefabRegistry` 中 `belongingKind=Daily` 的模板；gates 后续接 REQ-PRG / REQ-UNL | 场景已配置 `daily01`~`daily03` |
| 池为空 UX | Console `LogWarning`：「本日无新日常委托…」；后续可接终端邮件 | 已实现日志；邮件文案待 MSG |

> 数值平衡、剧情 gates 与地点随机（REQ-ASG-012）不在本首版定案范围；变更须升版本文并更新追踪表。
