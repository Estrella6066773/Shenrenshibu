# 玩法与需求阅读指引

| 字段 | 内容 |
|------|------|
| 最后更新 | 2026-06-15 |

本指引按**内容类型**概述 **系统设计/** 内的文档：产品愿景、功能需求、迭代计划与术语规范。**不按读者身份分类。**

全库入口见 [../00-阅读指引.md](../00-阅读指引.md)。实现细节在 [程序设计/](../程序设计/README.md)；世界观与剧情在 [设定设计/](../设定设计/README.md)。

---

## 一、本库回答什么问题

系统设计描述**玩家能体验到什么、规则理应怎样、如何验收**。正文**不展开**类名、字段表、Unity 映射；需要时可写「见程序设计：…」并附链接。

## 三、需求文档怎么读

每篇 `需求-<模块>.md` 通常包含：

| 章节 | 含义 |
|------|------|
| 元数据 | 模块键、版本、状态、关联实现 |
| 背景与目标 | 解决什么问题 |
| 范围 | **纳入范围** / **不纳入范围**（边界说明） |
| 功能需求 | 带 ID 的条目与验收要点 |
| 策划待定项 | 尚未定案的口径 |
| 关联文档 | 链到实现篇、设定篇或配对需求 |

需求总表与模块键对照见 [需求总览.md](02-需求/需求总览.md)。与实现的成对关系见 [程序设计/文档配对索引.md](../程序设计/文档配对索引.md)。

## 四、按主题定位需求

| 主题 | 需求入口 |
|------|----------|
| 核心循环（委托、员工、压力、矛盾） | `需求-委托系统`、`需求-员工模型与标签Buff`、`需求-压力系统`、`需求-矛盾系统` |
| 消息、对话、剧情进度 | `需求-消息系统-邮件聊天日志`、`需求-对话播放系统`、`需求-剧情进度` |
| 经济与日历 | `需求-玩家经济与日历时钟` |
| 界面与导航 | `需求-底部导航`、`需求-界面与弹窗`、`需求-主菜单与场景流` |
| 地图与任务 | `需求-地图与任务布点`、`需求-日常委托刷新` |
| 存档与解锁 | `需求-存档与进度`、`需求-科室协助解锁` |
| 简历与证件照 | `需求-简历窗口与交互`、`需求-简历标签与数据规则`、`需求-证件照生成` |

完整列表以 [需求总览.md](02-需求/需求总览.md) 为准。
