# 导出与一致性检查

## 第一条　导出管线（占位）

游戏引擎、Wiki 或本地化工具链的字段映射与构建脚本，建议放在本目录下的子文件夹（例如 `导出管线/`，等工具链落地时再建），并与 [玩家投放/vx.x/manifest.yaml](../玩家投放/v0.2/manifest.yaml)（**当前**为 v0.2）对齐。

注册表与命名规则见 [管理规则/README.md](../管理规则/README.md)。

## 第二条　一致性检查清单（人工／半自动）

建议在每次升版前执行：

1. [管理规则/id-registry.yaml](../管理规则/id-registry.yaml) 里的 `id` 与 [真相/](../真相/) 各文件 YAML 头里的 `id` 一致；
2. [玩家投放/vx.x/manifest.yaml](../玩家投放/v0.2/manifest.yaml)（**当前**为 v0.2）所列 `id` 均存在于注册表；
3. [对外读本](../玩家投放/v0.2/对外读本/00-世界观导览-继承说明.md) 所链世界观正文（[v0.1](../玩家投放/v0.1/对外读本/00-世界观导览与阅读分层.md)）与 [01-人物导览](../玩家投放/v0.2/对外读本/01-人物导览-四科负责人.md) 所引制度陈述，均须有 [真相](../真相/README.md) 对应条文支撑，避免未经核对的口径漂移；
4. 已废弃条目若仍出现在 manifest 引用里，要报错或触发升版流程；
5. **内部链接**：`python 04-设定/工具/检查/check-links.py` 须退出码为 0。已配置钩子时，**仅在 `git commit` 且暂存区含 `04-设定/*.md` 时**运行一次（`--fix --stage-fixed --quiet`）；无后台常驻。可选编辑期监视（30 分钟无保存/提交后自动退出）：[工具/检查/watch-links-idle.py](../工具/检查/watch-links-idle.py)。安装钩子：[工具/检查/install-git-hook.ps1](../工具/检查/install-git-hook.ps1)。书写约定见 [管理规则/03-内部链接书写规则.md](../管理规则/03-内部链接书写规则.md)。

## 第三条　与剧情文稿的接口

从 [剧情文稿/](../剧情文稿/README.md) 导出到游戏内时，应附带 `canon_refs` 列表（字段名可保留英文以利工具链），便于回归测试与考据者溯源（若产品策略允许公开溯源链）。
