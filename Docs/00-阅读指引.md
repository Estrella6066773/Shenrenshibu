# 文档阅读指引

| 字段 | 内容 |
|------|------|
| 最后更新 | 2026-06-15 |

本指引帮助参阅者按**内容类型**与**主题**理解文档库、定位条文，并掌握冲突时的优先级。**不按身份**（程序、策划、叙事等）分类；同一主题可能横跨多个目录，下文用「内容门类」串联入口。

全库结构概览见 [00-文档库总览.md](00-文档库总览.md)。仓库与 Unity 集成见 [00-工程集成说明.md](00-工程集成说明.md)。

---

## 一、文档库由什么组成

本地路径为 `神人事部/Docs/`。正文按**三分库**组织，另有**跨库规范**目录：

| 门类 | 路径 | 承载的内容 |
|------|------|------------|
| 跨库规范 | [00-规范/](00-规范/README.md) | 开放项汇总、三分库边界说明 |
| 系统设计 | [系统设计/](系统设计/README.md) | 玩家体验、玩法规则、验收口径 |
| 程序设计 | [程序设计/](程序设计/README.md) | 模块架构、数据结构、引擎映射 |
| 设定设计 | [设定设计/](设定设计/README.md) | 世界观真值、叙事、对外投放 |
| 文档模板 | [_templates/](_templates/) | 新建需求、实现等文档的模板 |

三分库各回答一类问题：**世界与故事是什么**（设定设计）、**玩起来理应怎样**（系统设计）、**具体如何实现**（程序设计）。同一主题常成对出现，索引见 [程序设计/文档配对索引.md](程序设计/文档配对索引.md)。

## 二、按内容类型查阅

### 1. 跨库治理与开放项

尚未定案、未实现或与条文有偏差的项，汇总在 [00-规范/待细化追踪.md](00-规范/待细化追踪.md)。已关闭项见 [待细化追踪-已完成.md](00-规范/待细化追踪-已完成.md)。

追踪表按**来源库**分节（跨库治理、系统设计、程序设计、设定设计），不按读者身份分节。

### 2. 世界观与设定真值

**设定真值**（内部权威母本）在 [设定设计/真相/](设定设计/真相/README.md)，含：

| 子类 | 路径 | 说明 |
|------|------|------|
| 世界基石 | `真相/00-基石/` | 纪年、治理秩序、空间模型 |
| 百科条目 | `真相/10-百科/` | 组织、地点、人物、系统、文化（以 `id` 为主键） |
| 叙事规范 | `真相/20-叙事规范/` | 母题、基调、终局分型、伦理红线（**不是**剧情细纲） |
| 主角与主线 | `真相/30-主角与主线/` | B-9002 视角的主线母本与开局流程 |
| 时序与历史 | `真相/40-时序与历史/` | 年表、城市母线、前任管理员背景 |

## 三、按主题快速定位

| 想查的内容 | 从这里看起 |
|------------|------------|
| 文档怎么组织、冲突听谁的 | 本文第二节、第四节 |
| 开放项与待对齐 | [待细化追踪.md](00-规范/待细化追踪.md) |
| 世界背景、组织、地点 | [设定设计/00-START-HERE.md](设定设计/00-START-HERE.md) → `真相/` |
| 剧情、主线、台词从哪找 | [设定设计/00-剧情阅读指引.md](设定设计/00-剧情阅读指引.md) |
| 某玩法模块的规则 | [需求总览.md](系统设计/02-需求/需求总览.md) → 对应 `需求-*` |
| 某模块怎么实现 | [实现总览.md](程序设计/04-实现/实现总览.md) → 对应 `实现-*` |
| 需求与实现是否对齐 | [文档配对索引.md](程序设计/文档配对索引.md)、[交付-追溯矩阵.md](程序设计/05-交付/交付-追溯矩阵.md) |
| 中文术语怎么写 | [词条库.md](系统设计/00-规范/词条库.md) |
| 对外已公布设定 | [玩家投放/](设定设计/玩家投放/README.md) 当前版本 |

可按主题跳读，**不必**从某一库的第一篇线性读到最后。

## 四、内容冲突时的优先级

| 情形 | 以谁为准 |
|------|----------|
| 叙事生产稿 vs 设定真值 | `真相/` |
| 对外读本 vs 内部真值 | 对外以 `玩家投放/` **当前版本**为准；内部修订先改 `真相/` 再升版投放 |
| 需求 vs 实现（文档阶段） | 以实现篇反映的**已定稿**行为为准；偏差记入 [待细化追踪](00-规范/待细化追踪.md) |
| 玩法/UI 中文名 vs 设定专名 | 机制与界面 → [词条库](系统设计/00-规范/词条库.md)；世界观专名 → `真相/` |
| `.草稿-*` 与正式树 | `.草稿-*` 仅供溯源，**不作**争议终局依据 |

## 五、常见文件名与类型

| 前缀 / 模式 | 含义 | 所在库 |
|-------------|------|--------|
| `需求-<模块>.md` | 功能与验收需求 | 系统设计 |
| `实现-<模块>.md` | 数据结构与判定逻辑 | 程序设计 |
| `交付-*` | 追溯、UAT、验收清单 | 程序设计 |
| `WLD-*`、`ORG-*`、`CHR-*`、`SYS-*`、`TIM-*` | 设定真值条目 `id` | 设定设计 / `真相/` |
| `NAR-*` | 叙事规范 | 设定设计 / `真相/20-叙事规范/` |
| `PCM-*` | 主角与主线母本 | 设定设计 / `真相/30-主角与主线/` |

文首元数据中的 `关联设计`、`canon_refs` 等字段用于跨库溯源；程序标识（如 `Assignment`、`Employee`）保留英文，旁边说明用词条库中文名。
