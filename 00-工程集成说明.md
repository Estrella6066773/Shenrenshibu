# Unity 工程集成说明

《神人事部》Unity 工程是本文档库的**实现载体**；**Git 仅在本仓库**（`ShenrenshibuStoryLib`），不是 Unity 根目录的第二套仓库。

## 角色划分

| 载体 | 版本控制 | 职责 |
|------|----------|------|
| **ShenrenshibuStoryLib（本仓库）** | Git → GitHub | 系统设计、程序设计、叙事正文、治理与 CI |
| **神人事部（Unity 根目录）** | UVC | `Assets/` 代码与数据、`ProjectSettings/` |

## 本地路径

文档库检出位置（相对 Unity 工程根）：

```
神人事部/Docs/ShenrenshibuStoryLib/    ← 本仓库根（在此执行 git）
```

## 克隆文档库

若只有 Unity 工程、尚无文档目录：

```bash
git clone git@github.com:Estrella6066773/ShenrenshibuStoryLib.git Docs/ShenrenshibuStoryLib
```

## 与 UVC 的分工（对齐机魂习惯）

- **Git**：`Docs/ShenrenshibuStoryLib/` 内全部 Markdown 与配置  
- **UVC**：Unity 根下的 `Assets/`、`ProjectSettings/`  

Unity 根目录 `ignore.conf` 已排除 `Docs/ShenrenshibuStoryLib/.git`，避免 UVC 与 Git 冲突。

## 改文档后的习惯

1. 在 `Docs/ShenrenshibuStoryLib/` 提交并 `git push origin main`。  
2. 若改行为：同步 `系统设计/02-需求` 与 `程序设计/04-实现`，并更新 `程序设计/05-交付/交付-追溯矩阵.md`。  
3. 更新「最后更新」前运行：`powershell -NoProfile -File Scripts/Get-DocsDate.ps1`（在本仓库根目录）。
