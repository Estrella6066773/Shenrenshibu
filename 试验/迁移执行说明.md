# 自 `正文/` 迁入本结构：执行说明

本文供评审通过后执行物理迁移时使用；迁移前**真值仍以 `正文/` 为准**。

## 前置条件

- `试验` 目录下规范与对照表已通过主笔与制作人确认。
- 已通知全体协作者暂停大规模重命名，或约定迁移窗口期。

## 推荐命令思路（Git）

在仓库根目录使用 `git mv` 保留历史，按域移动至 `试验/docs/20-设定真值/` 下对应子文件夹。移动完成后：

1. 全文检索替换旧路径字符串 `正文/01-实体域` 等为 `试验/docs/20-设定真值/实体域` 等（以实际目标为准）。
2. 更新 `词条来源对照表.md`、`ID注册表.md` 中的「现行正文路径」列。
3. 更新 `试验/README.md` 中「本目录不替代正文」一句为「真值宿主已迁至 `试验/docs/20-设定真值`」。
4. 原 `正文/` 可保留为空壳索引指向新路径，或删除后仅保留重定向说明文件（二选一，须统一）。

## 迁移后检查

- 任意 `LORE-*` 在仓库内唯一宿主文件。
- 相对链接与 Obsidian 等工具图谱无大面积断裂。
- `40-玩家投放包` 内 `_投放清单.yaml` 路径与实际文件一致。

## 回滚

迁移若以分支进行，回滚即恢复分支指针；若在主干进行，须依赖 Git  revert 或迁移前 tag。
