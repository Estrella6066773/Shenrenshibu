# 协作禁令与口径

本文面向策划、叙事、程序与外包接口人，列明**不可逾越**的协作规则。违反下列条款视为设定事故风险，须在评审中显式记录补救措施。

## 一、单一真值源

- **世界公理与终局约束**以 `LORE-RULE-001` 宿主文件为唯一增删入口。
- **全球史分期**以 `LORE-TIME-002` 为准；**指定城市史与三群体结构**以 `LORE-TIME-003` 为准；不得在 `LORE-RULE-002` 中写入与之冲突的长篇真值。
- **人事部四科与职权枚举**以 `LORE-ENT-ORG-001` 为准；科际控制与冲突链以 `LORE-REL-001` 为准；不得在观测词条中发明新的科室权力。

## 二、观测域不造设定

观测域条目仅归档**可验证现象、指标、舆情或制度外化后果**，且须标注上游依赖的实体、关系、规则或时序标识。  
若需新增身份类别、法律条文或组织结构，须在对应域新建或修订词条，再回写观测后果。

## 三、对外投放分层

- **`LORE-RULE-002`**：内部导航与禁令；默认不进入玩家资源包。
- **`LORE-RULE-003`**：作者向参照；**禁止**作为玩家可见设定原文。
- **玩家可见文案**：优先采信观测域已登记现象模板，并反向溯源；完整对外叙事应由 `试验/docs/40-玩家投放包/` 下列版本目录承载或经其登记。

## 四、叙事隐喻与公理冲突

任何叙事隐喻、社会学类比或语气参照，不得与 `LORE-RULE-001` 中公理级条文及置信度为 A 或 S 的时序、实体陈述相冲突。存在张力时须在相关词条「争议与冲突」节登记，并视需要写入 `试验/docs/50-治理与工程/变更裁定记录.md`。

## 五、标识与依赖

- 新建词条须先在 `试验/docs/50-治理与工程/ID注册表.md` 占用标识，避免与他人并行编辑冲突。
- 修改已冻结（`frozen`）词条须走书面评审与版本记录；不得在无记录情况下覆盖历史条文。

## 六、路径与版本控制

- 仓库内链接与文档路径**统一使用正斜杠**（`正文/03-规则域/...`），避免同一目录在版本控制中呈现为两套路径。
- `.草稿-ai不准修改` 等草稿区内容不得被宣发或程序直接引用为真值；入库须经过对应域词条与元数据校验。
