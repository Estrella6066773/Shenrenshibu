# 命名与文件路径约定

## 词条标识

- 格式：`LORE-<域代码>-<类型代码>-<三位序号>`
- **域代码**：`ENT`（实体）、`REL`（关系）、`RULE`（规则）、`TIME`（时序）、`OBS`（观测）。
- **类型代码**（实体域示例）：`CHR`（人物）、`ORG`（组织）、`LOC`（地点）、`RES`（资源）、`TEC`（技术）、`EVT`（事件）。其他域在首次占用时在 `ID注册表.md` 中声明类型代码含义。
- **序号**：三位十进制数字，同一 `(域, 类型)` 下递增；已占用序号不得复用。

标识一经发布即与**文件物理路径弱绑定**：允许整库迁移改路径，但不得改标识，除非走弃用与替代链路。

## 文件名

推荐两种形式，全库择一贯彻：

1. **标识优先**：`LORE-ENT-ORG-001.md`（最利于脚本检索与排序）。
2. **标识加短英文 slug**：`LORE-ENT-ORG-001_renshibu.md`（便于人类在目录中扫视）。

中文标题仅出现在文件内一级标题与 YAML 的 `title` 字段，**不作为**文件名唯一依据，以避免编码与排序差异。

## 路径书写

- 文档内相互引用、索引表、对照表，一律使用从仓库根目录起的相对路径，分隔符为 `/`。
- 新建目录名：优先使用**中文语义清晰**的文件夹名（如 `实体域/人物/`），与现行 `正文/` 风格兼容，降低迁移认知成本。若引入英文 slug，须在 `试验/docs/20-设定真值/_总索引.md` 中维护中英对照。

## 与程序或本地化对接的预留

- 对外包、表格导出或键值引用时，以 `id` 字段为主键；路径仅作人类可读属性。
- 多语言版本若日后引入，建议平行目录 `locales/zh-Hans/` 等，不在本阶段强制创建；但新增对外句段时应避免将不可翻译梗写死在与标识耦合的键名中。
