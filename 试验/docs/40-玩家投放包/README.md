# 玩家投放包

## 定位

本目录收纳**面向玩家、市场或外包可读**的设定切片，按**版本号**与**子渠道**组织。其功能是从内部真值中**解耦**对外表述：程序、宣发与本地化应优先引用本目录已登记文件，而非直接截取 `LORE-RULE-002` 全文或作者向隐喻条。

## 与内部词条的关系

- 对外句段须在文末或元数据中标注溯源标识（`LORE-*`），便于审计与修订。
- **`LORE-RULE-002`**：内部导航与禁令；默认不进入玩家包。
- **`LORE-RULE-003`**：**禁止**作为玩家设定原文。
- **现象层文案**：优先自 `LORE-OBS-*` 改写，并保留与上游条目的映射。

## 目录组织

```text
40-玩家投放包/
├── README.md                 （本文件）
├── v0.1-草案/                （示例版本；名称可按产品里程碑调整）
│   ├── _投放清单.yaml        （本包包含的标识、语言、生效范围）
│   ├── 共通/                 （全玩家可见的短背景）
│   ├── 阵营或身份切片/       （按需增删）
│   └── 系统与界面/           （加载提示、帮助文案等）
└── …
```

## 发布流程（摘要）

1. 策划在对应版本目录起草或摘录对外文案。  
2. 叙事负责人核对溯源与禁忌（对照 `试验/docs/00-入职与导读/协作禁令与口径.md`）。  
3. 更新 `_投放清单.yaml` 中的标识列表与 `visibility` 口径。  
4. 程序或宣发仅从清单列路径取数；若需新增出口，须先改清单再引用。
