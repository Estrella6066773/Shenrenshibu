# 可见性投放总约定

本文整合 `正文/90-可见性投放/00-索引.md` 中的出口规则，并上升为 `试验` 架构下的**总约定**。执行对外内容时，须同时满足本词条与宿主词条中的 `[投放注记]`。

## 基本原则

- **事实真值**写在对象五域词条正文；**谁能读到何种口径**由内嵌 `[投放注记]` 与词条 YAML `visibility` 描述。
- 不得为同一事实维护两套相互独立的正文；禁止「对外版」在无标识的情况下覆盖「内部版」真值。

## 按标识的出口等级

| 标识 | 玩家包默认 | 说明 |
|------|------------|------|
| `LORE-RULE-001` | 部分或间接 | 公理与术语可经改写进入教学或档案；长文一般不直接投给轻量用户 |
| `LORE-RULE-002` | **排除** | 内部导航与禁令；YAML 中 `visibility.player` 宜为 `none` |
| `LORE-RULE-003` | **禁止** | 作者向参照；不得作为玩家设定原文 |
| `LORE-OBS-*` | **优先采信** | 现象层与 UI 文案的优先来源；须反向溯源，不新增设定 |
| 其他实体、关系、时序 | 视注记与清单 | 须在 `40-玩家投放包` 对应版本 `_投放清单.yaml` 中显式列出后方可大规模引用 |

## 客户端与宣发接口

- 外包与宣发索取设定时，应提供**版本化包路径**（例如 `试验/docs/40-玩家投放包/v0.1-草案/`）及清单文件，而非直接打包整个 `正文/`。
- 若营销必须使用高风险概括语（例如「机器人管人」），须同步提供 `LORE-RULE-002` 中已写明的边界说明的**对外简写版**，并记入投放包。

## 与索引文件的关系

迁移完成前，词条内注记与 `正文/90-可见性投放/00-索引.md` 须保持同步；迁移完成后，可将该索引改为指向本文件及 `40-玩家投放包/README.md`。
