# 企业辖区的空间分层模型

## 第一条　模型之目的

本条文界定**第九区内部**城市空间之**分层叙事模型**，用于统一任务发生地、监控密度、法治外观与灰色经济之逻辑，避免「同一街区在不同文本中自相矛盾」。

须明确：本条文中的 A/B/C 分层为**人事部**（**社区事务与临时人力调度部**，亦称**人类事务管理部**）与相关执行链路的内部工作划分，并非法定或对外公布之官方行政区划。

## 第二条　三层结构

在地点设计尚未敲定前，文内出现之「A 区 / B 区 / C 区」仅为**明确地点称谓时的占位符**，用于指代对应分层，不等于正式地名。

### A 层：明亮层（企业橱窗区）

监控与执法无人机之部署密度高，消费与医疗资源在宣传口径上充足，对外构成**秩序与繁荣之橱窗**。叙事上可用于：体面交易背后之肮脏、舆论焦点、企业形象工程。

### B 层：缝隙层（居住与灰业混合区）

普通住户、小商贩与外包劳工混居，规则弹性大，冲突可通过谈判、贿赂或暴力等多路径解决。为**委托任务、社区纠纷与势力渗透**发生之主要场域。

### C 层：暗面层（维护通道、废弃区与黑市网络）

基础设施老化，系统覆盖存在断层，非法实验、黑市集散与隐藏事故遗留地集中。叙事上可用于：帮派据点、反抗组织藏匿、被企业刻意淡化之事件现场。

## 第三条　关键公共空间（制度—叙事接口）

下列地点为辖区内反复出现之**公共空间类型**；具体地理命名可由项目再行赋值，但功能类型须与本条一致：

| 类型 | 叙事功能概要 |
|------|----------------|
| 契约集中签署节点 | 劳务、债务、缓刑劳动等契约之集中签署；与「人类签字背锅」机制强相关。 |
| 暗面连通维修节点 | 连接 C 层之物理接口（井口、通风塔、维护廊道等）；追踪、伏击、藏匿之典型场景。 |
| 数据质押与隐私贴现节点 | 以隐私、记忆片段或账号权益换取流动性；黑客、舆情与勒索线之枢纽。 |
| 自动化医疗灰接口 | 自动化医疗节点；伪造病历、药品走私与人体改造等灰线之接口。 |
| 舆情宣传放大节点 | 巨型屏幕投放企业口号与关键绩效指标；舆情事件与群体骚乱之放大器。 |

除已定名之事变遗留片区 **废墟区**（见 [PLC-RUINS](../10-百科/地点/PLC-RUINS-废墟区.md)）外，上述名称均为**叙事功能类型**，不等于已定名地标；其余具体地名仍可再行赋值。
