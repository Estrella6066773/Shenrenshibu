# 辖区空间分层

## 第一条　适用范围与功能

本条文约定**第九区内**城市空间的**分层叙事模型**，用于统一书写任务发生地、监控密度、「法治」观感与灰色经济，避免同一街区在不同稿件中互相矛盾。

说明：文中的 A/B/C 分层是**人类事务管理部**（通称人事部）及其执行链条上的**内部分工用语**，不是法定或对外公布的行政区划。

## 第二条　三层结构

在街区尚未正式定名前，稿件中出现的「A 区 / B 区 / C 区」仅作为**区分层次的临时称谓**，不等于正式地名。

### A 层：明亮层（企业橱窗区）

监控与执法无人机部署密度较高，消费与医疗资源在宣传口径上显得充足，对外构成**秩序与繁荣的橱窗**。叙事上适合承载：体面交易之下的灰线、舆论焦点及企业形象工程。

### B 层：缝隙层（居住与灰产混杂区）

普通住户、小商贩与外包劳工混居，规则弹性较大，冲突可通过谈判、贿赂或暴力等多种路径展开。**委托任务、社区纠纷与势力渗透**多集中于此。

### C 层：暗面层（维修通道、废弃区与黑市网络）

基础设施老化，系统覆盖存在断层，非法实验、黑市集散与事故遗留地集中。叙事上适合承载：帮派据点、反抗组织的藏匿处，以及企业有意淡化的事故现场。

## 第三条　公共空间类型（制度—叙事接口）

下表列出辖区内会反复出现的**公共空间类型**；具体地名可由项目另行命名，但功能类型要与本表一致。

| 类型 | 叙事功能概要 |
|------|----------------|
| 契约集中签署节点 | 劳务、债务、缓刑劳动等契约集中签署；与「以人类签字承接可归责后果」的机制密切相关。 |
| 暗面连通维修节点 | 通向 C 层的物理通道（井口、通风塔、维修廊道等）；便于追踪、伏击与藏匿。 |
| 数据质押与隐私贴现节点 | 以隐私、记忆片段或账号权益换取流动性；黑客、舆情与勒索线的枢纽。 |
| 自动化医疗灰接口 | 自动化医疗节点上的灰色接口；伪造病历、药品走私与人体改造等叙事可由此接入。 |
| 舆情宣传放大节点 | 巨型屏幕滚动企业口号与 KPI；可作为舆情事件与群体骚乱的放大器。 |

除了已定名的 **废墟区**（见 [PLC-RUINS](../10-百科/地点/PLC-RUINS-废墟区-第九区事变遗留片区.md)）以外，上表名称都是**叙事功能类型**，并不是已敲定的地标名称；其余地名可在后续版本中赋值。
